# Kubernetes监控指标体系详解

## 监控指标分类

Kubernetes监控指标可以分为四个层次：

```
┌─────────────────────────────────────────────────────────────┐
│                     监控指标层次                             │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              应用层指标                              │    │
│  │         业务指标、应用性能指标                       │    │
│  └─────────────────────────────────────────────────────┘    │
│                            ↑                                 │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              Kubernetes资源指标                      │    │
│  │      Pod、Deployment、Service状态                    │    │
│  └─────────────────────────────────────────────────────┘    │
│                            ↑                                 │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              容器指标                                │    │
│  │      CPU、内存、网络、磁盘                           │    │
│  └─────────────────────────────────────────────────────┘    │
│                            ↑                                 │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              节点指标                                │    │
│  │      节点资源、系统负载                             │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 节点级指标

### CPU指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| node_cpu_seconds_total | CPU使用时间（各模式） | 秒 |
| node_load1 | 1分钟平均负载 | 数值 |
| node_load5 | 5分钟平均负载 | 数值 |
| node_load15 | 15分钟平均负载 | 数值 |

**常用查询**：

```promql
node_load5 / on(node) count(node_cpu_seconds_total) by (mode)
```

### 内存指标

| 指标名称 | 说明 |
|----------|------|
| node_memory_MemTotal_bytes | 总内存 |
| node_memory_MemAvailable_bytes | 可用内存 |
| node_memory_MemFree_bytes | 空闲内存 |
| node_memory_Buffers_bytes | 缓冲区内存 |
| node_memory_Cached_bytes | 缓存内存 |

**内存使用率计算**：

```promql
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100
```

### 磁盘指标

| 指标名称 | 说明 |
|----------|------|
| node_filesystem_size_bytes | 文件系统总大小 |
| node_filesystem_avail_bytes | 文件系统可用空间 |
| node_filesystem_files_free | 可用inode数 |
| node_disk_read_bytes_total | 磁盘读取字节数 |
| node_disk_written_bytes_total | 磁盘写入字节数 |

**磁盘使用率计算**：

```promql
(1 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"})) * 100
```

### 网络指标

| 指标名称 | 说明 |
|----------|------|
| node_network_receive_bytes_total | 网络接收字节数 |
| node_network_transmit_bytes_total | 网络发送字节数 |
| node_network_receive_drop_total | 接收丢包数 |
| node_network_transmit_drop_total | 发送丢包数 |

**网络流量速率**：

```promql
rate(node_network_receive_bytes_total[5m]) * 8
```

## 容器级指标

### CPU指标

| 指标名称 | 说明 |
|----------|------|
| container_cpu_usage_seconds_total | CPU使用时间 |
| container_cpu_cfs_periods_total | CFS调度周期数 |
| container_cpu_cfs_throttled_periods_total | 被限流的周期数 |
| container_cpu_cfs_throttled_seconds_total | 被限流的时间 |

**容器CPU使用率**：

```promql
rate(container_cpu_usage_seconds_total{container!=""}[5m]) * 100
```

**CPU限流比例**：

```promql
rate(container_cpu_cfs_throttled_seconds_total[5m]) / rate(container_cpu_cfs_periods_total[5m]) * 100
```

### 内存指标

| 指标名称 | 说明 |
|----------|------|
| container_memory_usage_bytes | 内存使用量 |
| container_memory_working_set_bytes | 工作集内存 |
| container_memory_cache | 缓存内存 |
| container_memory_rss | RSS内存 |
| container_memory_failcnt | 内存分配失败次数 |

**容器内存使用率**：

```promql
container_memory_working_set_bytes / container_spec_memory_limit_bytes * 100
```

### 网络指标

| 指标名称 | 说明 |
|----------|------|
| container_network_receive_bytes_total | 网络接收字节数 |
| container_network_transmit_bytes_total | 网络发送字节数 |
| container_network_receive_packets_total | 接收包数 |
| container_network_transmit_packets_total | 发送包数 |

## Kubernetes资源指标

### Pod指标

| 指标名称 | 说明 |
|----------|------|
| kube_pod_info | Pod信息 |
| kube_pod_status_phase | Pod状态 |
| kube_pod_status_ready | Pod就绪状态 |
| kube_pod_container_status_ready | 容器就绪状态 |
| kube_pod_container_status_restarts_total | 容器重启次数 |
| kube_pod_container_status_terminated_reason | 容器终止原因 |
| kube_pod_container_resource_requests | 资源请求 |
| kube_pod_container_resource_limits | 资源限制 |

**Pod状态分布**：

```promql
sum by (phase) (kube_pod_status_phase)
```

**重启次数最多的Pod**：

```promql
topk(10, sum by (pod, namespace) (increase(kube_pod_container_status_restarts_total[1h])))
```

### Deployment指标

| 指标名称 | 说明 |
|----------|------|
| kube_deployment_status_replicas | 当前副本数 |
| kube_deployment_status_replicas_available | 可用副本数 |
| kube_deployment_status_replicas_unavailable | 不可用副本数 |
| kube_deployment_spec_replicas | 期望副本数 |
| kube_deployment_status_replicas_updated | 已更新副本数 |

**Deployment副本数异常**：

```promql
kube_deployment_spec_replicas != kube_deployment_status_replicas_available
```

### Node指标

| 指标名称 | 说明 |
|----------|------|
| kube_node_info | 节点信息 |
| kube_node_status_condition | 节点状态条件 |
| kube_node_status_capacity | 节点资源容量 |
| kube_node_status_allocatable | 节点可分配资源 |
| kube_node_spec_unschedulable | 是否不可调度 |

**节点状态监控**：

```promql
kube_node_status_condition{condition="Ready", status!="true"}
```

### Service指标

| 指标名称 | 说明 |
|----------|------|
| kube_service_info | Service信息 |
| kube_service_spec_type | Service类型 |
| kube_service_spec_cluster_ip | ClusterIP |

| 指标名称 | 说明 |
|----------|------|
| kube_persistentvolumeclaim_info | PVC信息 |
| kube_persistentvolumeclaim_status_phase | PVC状态 |

| kubelet_volume_stats_used_bytes | 卷使用量 |

## 常用监控告警规则

### 节点告警

```yaml
groups:
- name: node
  rules:
  - alert: NodeHighCPU
    expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "节点CPU使用率过高"

  - alert: NodeHighMemory
    expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100 > 85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "节点内存使用率过高"

  - alert: NodeDiskPressure
    expr: (1 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"})) * 100 > 85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "节点磁盘空间不足"
```

### Pod告警

```yaml
groups:
- name: pod
  rules:
  - alert: PodCrashLooping
    expr: rate(kube_pod_container_status_restarts_total[15m]) * 60 * 15 > 0
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "Pod频繁重启"

  - alert: PodNotReady
    expr: kube_pod_status_phase{phase=~"Pending|Unknown"} > 0
    for: 10m
    labels:
      severity: warning
    annotations:
      summary: "Pod长时间未就绪"

  - alert: PodHighCPU
    expr: rate(container_cpu_usage_seconds_total{container!=""}[5m]) > 0.8
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "容器CPU使用率过高"
```

### 集群告警

```yaml
groups:
- name: cluster
  rules:
  - alert: NodeNotReady
    expr: kube_node_status_condition{condition="Ready", status!="true"} == 1
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "节点不可用"

  - alert: DeploymentReplicasMismatch
    expr: kube_deployment_spec_replicas != kube_deployment_status_replicas_available
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "Deployment副本数异常"

  - alert: PVCNearlyFull
    expr: kubelet_volume_stats_used_bytes / kubelet_volume_stats_capacity_bytes > 0.85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "PVC空间不足"
```

## 指标采集最佳实践

### 1. 分层监控

- 节点层：Node Exporter
- 容器层：cAdvisor
- 资源层：kube-state-metrics
- 应用层：应用自定义指标

### 2. 合理设置采集间隔

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s
```

### 3. 数据保留策略

```yaml
retention: 15d
storage:
  tsdb:
    retention.time: 15d
```

### 4. 使用Recording Rules

```yaml
groups:
- name: recording_rules
  rules:
  - record: node:cpu:usage
    expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)
  
  - record: node:memory:usage
    expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100
```
