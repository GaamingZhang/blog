# 自建Kubernetes集群完整步骤

## 为什么需要自建K8S集群？

在生产环境中，很多企业选择使用云厂商提供的托管Kubernetes服务（如EKS、AKS、GKE），但自建K8S集群仍然有其价值：

- **成本控制**：对于大规模集群，自建可能更经济
- **数据合规**：某些行业要求数据必须在本地数据中心
- **技术掌控**：深入理解K8S架构，便于问题排查
- **定制化需求**：可以根据业务需求深度定制

本文将详细介绍从零开始搭建一个生产级Kubernetes集群的完整步骤。

## 集群架构规划

在开始之前，需要规划集群架构。一个典型的生产集群包含：

### 节点角色

| 角色 | 数量 | 说明 |
|------|------|------|
| Master节点 | 3（奇数） | 运行控制平面组件，高可用需要至少3个 |
| Worker节点 | N | 运行工作负载，根据业务需求确定数量 |
| ETCD节点 | 3或5（奇数） | 可与Master共部署，也可独立部署 |

### 网络规划

```
集群网络: 10.244.0.0/16        # Pod网络
Service网络: 10.96.0.0/12      # Service ClusterIP网络
节点网络: 192.168.1.0/24       # 物理节点网络
```

**重要**：这三个网络不能重叠，否则会导致网络问题。

## 第一步：系统准备

### 1.1 操作系统要求

推荐使用Linux发行版：
- Ubuntu 20.04/22.04 LTS
- CentOS 7/8
- Rocky Linux 8/9

### 1.2 基础配置（所有节点执行）

```bash
# 设置主机名
hostnamectl set-hostname master1.example.com

# 配置hosts解析
cat >> /etc/hosts << EOF
192.168.1.10 master1
192.168.1.11 master2
192.168.1.12 master3
192.168.1.20 worker1
192.168.1.21 worker2
EOF

# 关闭swap
swapoff -a
sed -i '/swap/d' /etc/fstab

# 关闭防火墙（或配置规则）
systemctl stop firewalld
systemctl disable firewalld

# 关闭SELinux
setenforce 0
sed -i 's/^SELINUX=enforcing$/SELINUX=disabled/' /etc/selinux/config
```

### 1.3 内核参数配置

```bash
# 加载必要的内核模块
cat <<EOF | tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

modprobe overlay
modprobe br_netfilter

# 配置内核参数
cat <<EOF | tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sysctl --system
```

### 1.4 时间同步

```bash
# 安装并配置chrony
yum install -y chrony   # CentOS/RHEL
apt install -y chrony   # Ubuntu

systemctl enable chronyd
systemctl start chronyd

# 验证时间同步
chronyc sources
```

## 第二步：安装容器运行时

Kubernetes需要容器运行时（Container Runtime）来运行容器。推荐使用containerd。

### 2.1 安装containerd

```bash
# 安装依赖
yum install -y yum-utils device-mapper-persistent-data lvm2

# 配置Docker源
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装containerd
yum install -y containerd.io

# 生成默认配置
mkdir -p /etc/containerd
containerd config default > /etc/containerd/config.toml

# 修改配置，使用systemd cgroup驱动
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml

# 启动服务
systemctl enable containerd
systemctl start containerd
```

### 2.2 配置镜像加速（可选但推荐）

```bash
# 编辑 /etc/containerd/config.toml
[plugins."io.containerd.grpc.v1.cri".registry.mirrors]
  [plugins."io.containerd.grpc.v1.cri".registry.mirrors."docker.io"]
    endpoint = ["https://registry.docker-cn.com"]
```

## 第三步：安装kubeadm、kubelet、kubectl

### 3.1 配置Kubernetes源

```bash
# CentOS/RHEL
cat <<EOF | tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://packages.cloud.google.com/yum/repos/kubernetes-el7-\$basearch
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://packages.cloud.google.com/yum/doc/yum-key.gpg https://packages.cloud.google.com/yum/doc/rpm-package-key.gpg
EOF

# 国内可使用阿里云镜像
cat <<EOF | tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64/
enabled=1
gpgcheck=0
EOF
```

### 3.2 安装组件

```bash
# 查看可用版本
yum list kubeadm --showduplicates | sort -r

# 安装指定版本（建议指定版本，避免版本不一致）
yum install -y kubeadm-1.28.0 kubelet-1.28.0 kubectl-1.28.0

# 锁定版本（防止自动升级）
yum versionlock add kubeadm kubelet kubectl

# 启动kubelet（此时会启动失败，正常现象）
systemctl enable kubelet
```

## 第四步：部署高可用ETCD集群

对于生产环境，建议部署独立的ETCD集群。

### 4.1 在ETCD节点上安装etcd

```bash
# 下载etcd
ETCD_VER=v3.5.9
wget https://github.com/etcd-io/etcd/releases/download/${ETCD_VER}/etcd-${ETCD_VER}-linux-amd64.tar.gz
tar xzvf etcd-${ETCD_VER}-linux-amd64.tar.gz
mv etcd-${ETCD_VER}-linux-amd64/etcd* /usr/local/bin/

# 创建配置目录
mkdir -p /etc/etcd /var/lib/etcd
```

### 4.2 创建ETCD配置文件

在etcd1节点：

```bash
cat > /etc/etcd/etcd.conf << EOF
ETCD_NAME=etcd1
ETCD_DATA_DIR=/var/lib/etcd
ETCD_LISTEN_PEER_URLS=https://192.168.1.30:2380
ETCD_LISTEN_CLIENT_URLS=https://192.168.1.30:2379,https://127.0.0.1:2379
ETCD_INITIAL_ADVERTISE_PEER_URLS=https://192.168.1.30:2380
ETCD_ADVERTISE_CLIENT_URLS=https://192.168.1.30:2379
ETCD_INITIAL_CLUSTER=etcd1=https://192.168.1.30:2380,etcd2=https://192.168.1.31:2380,etcd3=https://192.168.1.32:2380
ETCD_INITIAL_CLUSTER_TOKEN=etcd-cluster
ETCD_INITIAL_CLUSTER_STATE=new
ETCD_CERT_FILE=/etc/etcd/pki/server.crt
ETCD_KEY_FILE=/etc/etcd/pki/server.key
ETCD_TRUSTED_CA_FILE=/etc/etcd/pki/ca.crt
ETCD_PEER_CERT_FILE=/etc/etcd/pki/peer.crt
ETCD_PEER_KEY_FILE=/etc/etcd/pki/peer.key
ETCD_PEER_TRUSTED_CA_FILE=/etc/etcd/pki/ca.crt
EOF
```

### 4.3 创建systemd服务

```bash
cat > /etc/systemd/system/etcd.service << EOF
[Unit]
Description=etcd
After=network.target

[Service]
Type=notify
EnvironmentFile=/etc/etcd/etcd.conf
ExecStart=/usr/local/bin/etcd
Restart=always
RestartSec=10s

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable etcd
systemctl start etcd
```

### 4.4 验证ETCD集群

```bash
etcdctl --endpoints=https://192.168.1.30:2379,https://192.168.1.31:2379,https://192.168.1.32:2379 \
  --cacert=/etc/etcd/pki/ca.crt \
  --cert=/etc/etcd/pki/server.crt \
  --key=/etc/etcd/pki/server.key \
  endpoint health
```

## 第五步：初始化Master节点

### 5.1 创建kubeadm配置文件

```yaml
cat > kubeadm-config.yaml << EOF
apiVersion: kubeadm.k8s.io/v1beta3
kind: ClusterConfiguration
kubernetesVersion: v1.28.0
controlPlaneEndpoint: "k8s-api.example.com:6443"
networking:
  podSubnet: "10.244.0.0/16"
  serviceSubnet: "10.96.0.0/12"
etcd:
  external:
    endpoints:
    - https://192.168.1.30:2379
    - https://192.168.1.31:2379
    - https://192.168.1.32:2379
    caFile: /etc/etcd/pki/ca.crt
    certFile: /etc/kubernetes/pki/etcd/etcd.crt
    keyFile: /etc/kubernetes/pki/etcd/etcd.key
---
apiVersion: kubeadm.k8s.io/v1beta3
kind: InitConfiguration
localAPIEndpoint:
  advertiseAddress: 192.168.1.10
  bindPort: 6443
EOF
```

### 5.2 初始化第一个Master节点

```bash
# 预下载镜像
kubeadm config images pull --config kubeadm-config.yaml

# 初始化集群
kubeadm init --config kubeadm-config.yaml --upload-certs
```

### 5.3 配置kubectl

```bash
mkdir -p $HOME/.kube
cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
chown $(id -u):$(id -g) $HOME/.kube/config

# 验证
kubectl get nodes
```

## 第八步：安装网络插件

Kubernetes需要CNI网络插件才能让Pod之间通信。

### 8.1 安装Calico（推荐）

```bash
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.26.1/manifests/calico.yaml
```

### 8.2 或安装Flannel

```bash
kubectl apply -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml
```

### 8.3 验证网络

```bash
kubectl get pods -n kube-system | grep -E 'calico|flannel'
```

## 第九步：配置负载均衡器

生产环境需要为API Server配置负载均衡。

### 9.1 使用HAProxy + Keepalived

在负载均衡节点安装：

```bash
yum install -y haproxy keepalived
```

### 9.2 HAProxy配置

```haproxy
cat > /etc/haproxy/haproxy.cfg << EOF
global
    log /dev/log local0
    log /dev/log local1 notice
    chroot /var/lib/haproxy
    stats socket /var/lib/haproxy/stats mode 660
    stats timeout 30s
    user haproxy
    group haproxy
    daemon

defaults
    log     global
    mode    tcp
    option  tcplog
    option  dontlognull
    timeout connect 5000
    timeout client  50000
    timeout server  50000

frontend k8s-api
    bind *:6443
    mode tcp
    option tcplog
    default_backend k8s-api

backend k8s-api
    mode tcp
    option tcp-check
    balance roundrobin
    server master1 192.168.1.10:6443 check
    server master2 192.168.1.11:6443 check
    server master3 192.168.1.12:6443 check
EOF
```

### 9.3 Keepalived配置

```bash
cat > /etc/keepalived/keepalived.conf << EOF
global_defs {
    router_id LVS_DEVEL
}

vrrp_script check_haproxy {
    script "killall -0 haproxy"
    interval 3
    weight -2
    fall 10
    rise 2
}

vrrp_instance VI_1 {
    state MASTER
    interface eth0
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 42
    }
    virtual_ipaddress {
        192.168.1.100
    }
    track_script {
        check_haproxy
    }
}
EOF
```

## 常见问题

### Q1: kubeadm init失败，提示镜像拉取超时

解决方案：预下载镜像或配置镜像加速

```bash
kubeadm config images pull --image-repository registry.aliyuncs.com/google_containers
```

### Q2: 节点状态为NotReady

原因：网络插件未安装或未正常运行

```bash
kubectl get pods -n kube-system
kubectl describe node <node-name>
```

### Q3: kubelet启动失败

常见原因：
- swap未关闭
- cgroup驱动不匹配（containerd使用systemd，kubelet也要配置systemd）
- 防火墙阻止

### Q4: ETCD集群不健康

```bash
# 检查ETCD状态
etcdctl endpoint status --cluster -w table

# 检查ETCD日志
journalctl -u etcd -f
```

## 最佳实践

1. **版本一致性**：确保所有组件版本一致，避免兼容性问题
2. **证书管理**：定期更新证书，kubeadm默认证书有效期1年
3. **备份ETCD**：定期备份ETCD数据，这是集群恢复的关键
4. **资源规划**：Master节点建议至少4核8G，ETCD节点建议使用SSD
5. **网络隔离**：生产环境建议网络隔离，使用防火墙规则

## 总结

自建Kubernetes集群涉及以下核心步骤：

1. 系统准备（内核参数、时间同步、关闭swap）
2. 安装容器运行时（containerd）
3. 安装Kubernetes组件（kubeadm、kubelet、kubectl）
4. 部署ETCD集群（高可用）
5. 初始化Master节点
6. 加入其他节点
7. 安装网络插件
8. 配置负载均衡器
9. 验证集群

每个步骤都需要仔细配置，任何一个环节出错都可能导致集群无法正常工作。建议在生产环境部署前，先在测试环境完整演练一遍。
