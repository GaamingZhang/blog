# Kubernetes Pod 日志分析

## 引言

日志是排查 Kubernetes 应用问题的重要依据。通过查看和分析 Pod 日志，可以快速定位问题原因。本文介绍 Kubernetes Pod 日志的查看方法、日志收集方案和日志分析技巧。

## Pod 日志概述

### 日志类型

```
┌─────────────────────────────────────────────────────────────┐
│              Kubernetes 日志类型                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  标准输出日志（stdout）：                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 容器标准输出                                     │   │
│  │  • 由容器运行时收集                                 │   │
│  │  • 存储在 /var/log/containers                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  标准错误日志（stderr）：                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 容器标准错误                                     │   │
│  │  • 与标准输出一起收集                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  文件日志：                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 容器内部文件                                     │   │
│  │  • 需要挂载存储卷                                   │   │
│  │  • 需要日志采集器收集                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 查看 Pod 日志

### 基本命令

```bash
kubectl logs <pod-name>

kubectl logs <pod-name> -n <namespace>

kubectl logs <pod-name> -c <container-name>

kubectl logs <pod-name> --tail=100

kubectl logs <pod-name> --since=1h

kubectl logs <pod-name> -f
```

### 查看多容器 Pod 日志

```bash
kubectl logs <pod-name> -c <container-name>

kubectl logs <pod-name> --all-containers

kubectl logs <pod-name> -c init-container
```

### 查看之前容器的日志

```bash
kubectl logs <pod-name> --previous

kubectl logs <pod-name> -c <container-name> --previous
```

### 查看日志文件

```bash
kubectl exec <pod-name> -- cat /var/log/app.log

kubectl exec <pod-name> -- ls -la /var/log/
```

## 日志分析技巧

### 过滤日志

```bash
kubectl logs <pod-name> | grep "ERROR"

kubectl logs <pod-name> | grep -i "error\|warning"

kubectl logs <pod-name> | grep -v "DEBUG"
```

### 统计日志

```bash
kubectl logs <pod-name> | grep "ERROR" | wc -l

kubectl logs <pod-name> | awk '{print $1}' | sort | uniq -c
```

### 时间范围过滤

```bash
kubectl logs <pod-name> --since-time="2024-01-01T00:00:00Z"

kubectl logs <pod-name> --since=1h
```

## 日志收集方案

### EFK 方案

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  namespace: logging
spec:
  selector:
    matchLabels:
      app: fluentd
  template:
    metadata:
      labels:
        app: fluentd
    spec:
      containers:
      - name: fluentd
        image: fluent/fluentd-kubernetes-daemonset:v1-debian-elasticsearch
        env:
        - name: FLUENT_ELASTICSEARCH_HOST
          value: "elasticsearch.logging.svc.cluster.local"
        - name: FLUENT_ELASTICSEARCH_PORT
          value: "9200"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers
```

## 常见日志问题

### 日志过大

```bash
kubectl logs <pod-name> --tail=1000 > pod.log

kubectl exec <pod-name> -- du -sh /var/log/
```

### 日志丢失

```bash
kubectl describe pod <pod-name>

kubectl get events --field-selector involvedObject.name=<pod-name>
```

### 日志格式混乱

```bash
kubectl logs <pod-name> | jq .
```

## 最佳实践

### 1. 使用结构化日志

```json
{
  "timestamp": "2024-01-01T10:00:00Z",
  "level": "INFO",
  "message": "Request processed"
}
```

### 2. 设置日志级别

```yaml
env:
- name: LOG_LEVEL
  value: "info"
```

### 3. 日志轮转

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-log-rotation
spec:
  containers:
  - name: app
    image: myapp:v1
    volumeMounts:
    - name: logs
      mountPath: /var/log/app
  volumes:
  - name: logs
    emptyDir:
      sizeLimit: 1Gi
```

### 4. 关联追踪 ID

```yaml
env:
- name: TRACE_ID
  valueFrom:
    fieldRef:
      fieldPath: metadata.uid
```

## 面试回答

**问题**: 如何查看和分析 Pod 的日志？

**回答**: Kubernetes Pod 日志查看和分析方法：

**查看日志命令**：`kubectl logs <pod-name>` 查看当前日志；`kubectl logs <pod-name> -c <container>` 指定容器；`kubectl logs <pod-name> --tail=N` 查看最后 N 行；`kubectl logs <pod-name> --since=1h` 查看最近 1 小时日志；`kubectl logs <pod-name> -f` 实时跟踪日志；`kubectl logs <pod-name> --previous` 查看之前容器的日志。

**日志分析技巧**：使用 grep 过滤关键字（如 ERROR、WARNING）；使用 awk、sort、uniq 统计日志；使用 jq 解析 JSON 格式日志；结合时间范围过滤定位问题。

**日志类型**：**标准输出日志**由容器运行时自动收集，存储在 /var/log/containers；**文件日志**需要挂载存储卷，使用日志采集器收集。

**日志收集方案**：EFK（Elasticsearch + Fluentd + Kibana）是最常用的方案。Fluentd 以 DaemonSet 部署在每个节点，采集容器日志发送到 Elasticsearch，Kibana 提供可视化查询。

**最佳实践**：使用结构化日志（JSON 格式）；设置合理的日志级别；配置日志轮转避免磁盘满；添加追踪 ID 关联请求链路；使用日志聚合平台集中管理日志。
