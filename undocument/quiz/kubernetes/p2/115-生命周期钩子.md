# Kubernetes 生命周期钩子

## 引言

在 Kubernetes 中，容器的生命周期管理是应用可靠运行的关键。生命周期钩子（Lifecycle Hooks）允许在容器生命周期的特定时刻执行自定义操作，如启动前初始化、关闭前清理资源等。合理使用生命周期钩子可以确保应用的优雅启动和关闭。

## 生命周期钩子概述

### 容器生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                容器生命周期流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  容器创建                                                    │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              postStart 钩子                          │   │
│  │  • 容器创建后立即执行                                │   │
│  │  • 在 ENTRYPOINT 之前执行                            │   │
│  │  • 阻塞容器启动直到完成                              │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              主进程运行                              │   │
│  │  • 执行 ENTRYPOINT/CMD                              │   │
│  │  • 容器正常运行状态                                  │   │
│  │  • 健康检查（liveness/readiness）                   │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              preStop 钩子                            │   │
│  │  • 容器终止前执行                                    │   │
│  │  • 在 SIGTERM 之前执行                               │   │
│  │  • 用于优雅关闭                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                      │
│       ▼                                                      │
│  容器终止                                                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 钩子类型对比

| 钩子类型 | 执行时机 | 用途 | 阻塞性 |
|---------|---------|------|--------|
| postStart | 容器创建后 | 初始化、注册服务 | 是 |
| preStop | 容器终止前 | 清理资源、注销服务 | 是 |

## postStart 钩子

### 基本配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: poststart-demo
spec:
  containers:
  - name: app
    image: nginx:1.21
    lifecycle:
      postStart:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Container started at $(date)" >> /var/log/lifecycle.log
            mkdir -p /data/app
            chmod 755 /data/app
```

### postStart 使用场景

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-poststart
spec:
  containers:
  - name: app
    image: my-app:v1
    lifecycle:
      postStart:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Waiting for dependencies..."
            until nc -z mysql-service 3306; do
              sleep 1
            done
            echo "Dependencies ready, starting application"
            curl -X POST http://service-registry/register \
              -H "Content-Type: application/json" \
              -d '{"service":"my-app","port":8080}'
    env:
    - name: POD_IP
      valueFrom:
        fieldRef:
          fieldPath: status.podIP
```

### postStart 与 Init 容器对比

```
┌─────────────────────────────────────────────────────────────┐
│          postStart 与 Init 容器对比                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Init 容器：                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  执行顺序：Init 容器 -> postStart -> 主进程         │   │
│  │  独立镜像：可以使用不同的镜像                        │   │
│  │  失败处理：Pod 重启                                  │   │
│  │  适用场景：复杂初始化、依赖检查                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  postStart 钩子：                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  执行顺序：与主进程并行（但阻塞状态）                │   │
│  │  同一镜像：使用容器相同镜像                          │   │
│  │  失败处理：容器重启                                  │   │
│  │  适用场景：简单初始化、服务注册                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## preStop 钩子

### 基本配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: prestop-demo
spec:
  terminationGracePeriodSeconds: 60
  containers:
  - name: app
    image: nginx:1.21
    lifecycle:
      preStop:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Container stopping at $(date)" >> /var/log/lifecycle.log
            nginx -s quit
            sleep 10
```

### preStop 实现优雅关闭

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: graceful-shutdown
spec:
  terminationGracePeriodSeconds: 60
  containers:
  - name: app
    image: my-app:v1
    ports:
    - containerPort: 8080
    lifecycle:
      preStop:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Graceful shutdown initiated"
            curl -X POST http://localhost:8080/shutdown
            curl -X DELETE http://service-registry/deregister \
              -H "Content-Type: application/json" \
              -d '{"service":"my-app"}'
            sleep 15
            echo "Graceful shutdown completed"
```

### preStop 与 Service 配合

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      terminationGracePeriodSeconds: 60
      containers:
      - name: app
        image: my-app:v1
        ports:
        - containerPort: 8080
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 10
        lifecycle:
          preStop:
            exec:
              command:
              - /bin/sh
              - -c
              - |
                curl -X POST http://localhost:8080/drain
                sleep 20
```

## 钩子处理器类型

### exec 处理器

```yaml
lifecycle:
  postStart:
    exec:
      command:
      - /bin/sh
      - -c
      - echo "Post start hook executed"
  preStop:
    exec:
      command:
      - /bin/sh
      - -c
      - echo "Pre stop hook executed"
```

### httpGet 处理器

```yaml
lifecycle:
  postStart:
    httpGet:
      path: /initialize
      port: 8080
      httpHeaders:
      - name: Content-Type
        value: application/json
  preStop:
    httpGet:
      path: /shutdown
      port: 8080
```

### tcpSocket 处理器

```yaml
lifecycle:
  postStart:
    tcpSocket:
      port: 8080
```

## 完整生命周期示例

### Web 应用生命周期管理

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web-app
  template:
    metadata:
      labels:
        app: web-app
    spec:
      terminationGracePeriodSeconds: 60
      containers:
      - name: web
        image: nginx:1.21
        ports:
        - containerPort: 80
        env:
        - name: POD_NAME
          valueFrom:
            fieldRef:
              fieldPath: metadata.name
        - name: POD_IP
          valueFrom:
            fieldRef:
              fieldPath: status.podIP
        lifecycle:
          postStart:
            exec:
              command:
              - /bin/sh
              - -c
              - |
                echo "Pod ${POD_NAME} starting at $(date)" >> /var/log/lifecycle.log
                curl -X POST http://service-registry/register \
                  -H "Content-Type: application/json" \
                  -d '{"name":"'"${POD_NAME}"'","ip":"'"${POD_IP}"'","port":80}'
          preStop:
            exec:
              command:
              - /bin/sh
              - -c
              - |
                echo "Pod ${POD_NAME} stopping at $(date)" >> /var/log/lifecycle.log
                curl -X POST http://localhost/drain
                curl -X DELETE http://service-registry/deregister \
                  -H "Content-Type: application/json" \
                  -d '{"name":"'"${POD_NAME}"'"}'
                sleep 10
        readinessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 10
        livenessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 15
          periodSeconds: 20
```

### 数据库连接池应用

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: db-app
spec:
  terminationGracePeriodSeconds: 120
  containers:
  - name: app
    image: my-db-app:v1
    ports:
    - containerPort: 8080
    lifecycle:
      postStart:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Initializing database connection pool..."
            until nc -z mysql-service 3306; do
              echo "Waiting for MySQL..."
              sleep 2
            done
            echo "Database connection pool initialized"
      preStop:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            echo "Closing database connections..."
            curl -X POST http://localhost:8080/close-connections
            sleep 30
            echo "All connections closed"
```

## 钩子执行顺序

### Pod 启动顺序

```
┌─────────────────────────────────────────────────────────────┐
│                  Pod 启动顺序                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. Init 容器执行（如果有）                                  │
│       │                                                      │
│       ▼                                                      │
│  2. 主容器创建                                               │
│       │                                                      │
│       ▼                                                      │
│  3. postStart 钩子执行                                       │
│       │                                                      │
│       │  ┌─────────────────────────────────────────────┐   │
│       │  │  postStart 与 ENTRYPOINT 并行执行           │   │
│       │  │  但 postStart 必须完成后容器才进入运行状态  │   │
│       │  └─────────────────────────────────────────────┘   │
│       ▼                                                      │
│  4. ENTRYPOINT/CMD 执行                                      │
│       │                                                      │
│       ▼                                                      │
│  5. 容器进入运行状态                                         │
│       │                                                      │
│       ▼                                                      │
│  6. 健康检查开始                                             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Pod 终止顺序

```
┌─────────────────────────────────────────────────────────────┐
│                  Pod 终止顺序                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. Pod 标记为 Terminating                                   │
│       │                                                      │
│       ▼                                                      │
│  2. Service 移除 Pod 端点                                    │
│       │                                                      │
│       ▼                                                      │
│  3. preStop 钩子执行                                         │
│       │                                                      │
│       │  ┌─────────────────────────────────────────────┐   │
│       │  │  preStop 执行时间计入 terminationGracePeriod│   │
│       │  │  默认 30 秒，可自定义                       │   │
│       │  └─────────────────────────────────────────────┘   │
│       ▼                                                      │
│  4. 发送 SIGTERM 信号                                        │
│       │                                                      │
│       ▼                                                      │
│  5. 等待 terminationGracePeriodSeconds                      │
│       │                                                      │
│       ▼                                                      │
│  6. 发送 SIGKILL 信号强制终止                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 最佳实践

### 1. 设置合理的超时时间

```yaml
spec:
  terminationGracePeriodSeconds: 60
  containers:
  - name: app
    image: my-app:v1
    lifecycle:
      preStop:
        exec:
          command:
          - /bin/sh
          - -c
          - |
            timeout 45 /app/shutdown.sh || true
```

### 2. 幂等性设计

```yaml
lifecycle:
  postStart:
    exec:
      command:
      - /bin/sh
      - -c
      - |
        if [ ! -f /data/initialized ]; then
          echo "Initializing..."
          touch /data/initialized
        fi
```

### 3. 错误处理

```yaml
lifecycle:
  preStop:
    exec:
      command:
      - /bin/sh
      - -c
      - |
        curl -f http://localhost:8080/shutdown || true
        sleep 10
```

### 4. 日志记录

```yaml
lifecycle:
  postStart:
    exec:
      command:
      - /bin/sh
      - -c
      - |
        exec >> /var/log/lifecycle.log 2>&1
        echo "$(date): postStart hook started"
        echo "$(date): postStart hook completed"
```

## 面试回答

**问题**: 什么是 Kubernetes 的生命周期钩子？

**回答**: Kubernetes 生命周期钩子是容器生命周期管理的重要机制，允许在容器生命周期的特定时刻执行自定义操作。

**两种钩子类型**：**postStart** 在容器创建后立即执行，与 ENTRYPOINT 并行但阻塞容器进入运行状态，用于初始化操作如服务注册、依赖检查；**preStop** 在容器终止前执行，在发送 SIGTERM 之前调用，用于优雅关闭如注销服务、关闭连接、清理资源。

**钩子处理器**：支持三种处理器类型。**exec** 执行容器内命令，最常用；**httpGet** 发送 HTTP 请求到容器内服务；**tcpSocket** 检查 TCP 端口是否可用。

**执行顺序**：Pod 启动时，Init 容器 -> postStart 钩子 -> 主进程运行；Pod 终止时，preStop 钩子 -> SIGTERM -> 等待 terminationGracePeriodSeconds -> SIGKILL 强制终止。

**与 Init 容器区别**：postStart 与主容器共享镜像和文件系统，Init 容器使用独立镜像；postStart 与主进程并行执行，Init 容器顺序执行；复杂初始化推荐使用 Init 容器，简单操作使用 postStart。

**最佳实践**：设置合理的 terminationGracePeriodSeconds；钩子脚本设计为幂等性；添加错误处理避免钩子失败导致容器无法启动；记录日志便于调试；preStop 中添加 sleep 确保服务注销完成。
