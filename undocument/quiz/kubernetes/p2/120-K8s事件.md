# Kubernetes 事件

## 引言

在 Kubernetes 中，事件（Event）是记录集群中发生的重要操作的资源对象。事件提供了关于集群内部活动的详细信息，是排查问题、监控集群状态的重要数据来源。理解 Kubernetes 事件的机制和使用方法，对于运维和故障排查至关重要。

## 事件概述

### 事件的作用

```
┌─────────────────────────────────────────────────────────────┐
│                Kubernetes 事件作用                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │

│  │  1. 记录资源生命周期事件                             │   │

│  │     • Deployment 扩缩容、更新                        │   │
```

### 事件类型

| 事件类型 | 说明 |
|---------|------|
| Normal | 正常操作事件 |
| Warning | 警告事件，需要关注 |

### 常见事件原因

| 原因 | 说明 |
|-----|------|
| Scheduled | Pod 被调度到节点 |
| Pulled | 镜像拉取成功 |
| Created | 容器创建成功 |
| Started | 容器启动成功 |
| FailedScheduling | 调度失败 |
| FailedMount | 挂载卷失败 |
| BackOff | 容器启动失败退避 |
| OOMKilled | 内存超限被终止 |

## 查看事件

### 基本命令

```bash
kubectl get events

kubectl get events -n <namespace>

kubectl get events --sort-by='.lastTimestamp'

kubectl get events --field-selector type=Warning
```

### 查看 Pod 相关事件

```bash
kubectl describe pod <pod-name> -n <namespace>

kubectl get events --field-selector involvedObject.name=<pod-name>
```

### 查看节点相关事件

```bash
kubectl describe node <node-name>

kubectl get events --field-selector involvedObject.name=<node-name>
```

## 事件结构

### 事件字段说明

```yaml
apiVersion: v1
kind: Event
metadata:
  name: pod-event.17a2b3c4
  namespace: default
involvedObject:
  kind: Pod
  namespace: default
  name: my-pod
  uid: 12345678-1234-1234-1234-123456789012
  apiVersion: v1
reason: Scheduled
message: Successfully assigned default/my-pod to node-1
source:
  component: default-scheduler
firstTimestamp: "2024-01-01T10:00:00Z"
lastTimestamp: "2024-01-01T10:00:00Z"
count: 1
type: Normal
```

### 事件字段详解

| 字段 | 说明 |
|-----|------|
| involvedObject | 关联的资源对象 |
| reason | 事件原因 |
| message | 事件详细信息 |
| source | 事件来源组件 |
| firstTimestamp | 首次发生时间 |
| lastTimestamp | 最后发生时间 |
| count | 发生次数 |
| type | 事件类型（Normal/Warning） |

## 事件与故障排查

### Pod 启动失败排查

```bash
kubectl describe pod <pod-name>

kubectl get events --field-selector involvedObject.name=<pod-name>
```

### 调度失败排查

```bash
kubectl get events --field-selector reason=FailedScheduling

kubectl describe pod <pod-name> | grep -A 10 Events
```

### 资源不足排查

```bash
kubectl get events --field-selector type=Warning

kubectl describe nodes | grep -A 5 "Allocated resources"
```

## 面试回答

**问题**: 什么是 Kubernetes 的事件？

**回答**: Kubernetes 事件（Event）是记录集群中重要操作的资源对象，提供了关于集群内部活动的详细信息。

**事件类型**：**Normal** 类型记录正常操作，如 Pod 调度成功、容器启动成功；**Warning** 类型记录异常和警告，如镜像拉取失败、资源不足、健康检查失败。

**常见事件原因**：Scheduled（调度成功）、FailedScheduling（调度失败）、Pulled（镜像拉取成功）、ErrImagePull（镜像拉取失败）、Started（容器启动）、BackOff（容器启动失败退避）、OOMKilled（内存超限被终止）。

**查看事件**：使用 `kubectl get events` 查看所有事件；`kubectl describe pod <pod-name>` 查看 Pod 相关事件；`kubectl get events --field-selector type=Warning` 查看 Warning 事件；`kubectl get events --sort-by='.lastTimestamp'` 按时间排序。

**事件保留**：默认保留 1 小时，可通过 kube-apiserver 的 --event-ttl 参数调整。事件存储在 etcd 中，过长保留时间会增加存储压力。

**故障排查**：通过 describe 命令查看资源事件；关注 Warning 类型事件；按时间排序追踪问题时间线；使用事件定位 Pod 启动失败、调度失败、资源不足等问题。

**最佳实践**：定期检查 Warning 事件；使用外部系统持久化事件；结合日志和监控进行综合分析。
