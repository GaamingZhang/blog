# Kubernetes 容器安全管理

## 引言

容器安全是 Kubernetes 安全的重要组成部分。通过合理配置 Security Context、使用 Pod Security Standards、镜像安全扫描等措施，可以有效提升容器的安全性。本文介绍 Kubernetes 容器安全管理的方法和最佳实践。

## Security Context

### Pod 级别 Security Context

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    runAsGroup: 1000
    fsGroup: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    image: myapp:v1
```

### 容器级别 Security Context

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: container-security
spec:
  containers:
  - name: app
    image: myapp:v1
    securityContext:
      runAsNonRoot: true
      runAsUser: 1000
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
        add:
        - NET_BIND_SERVICE
    volumeMounts:
    - name: tmp
      mountPath: /tmp
  volumes:
  - name: tmp
    emptyDir: {}
```

### Security Context 参数

| 参数 | 说明 |
|-----|------|
| runAsUser | 以指定用户 ID 运行 |
| runAsGroup | 以指定组 ID 运行 |
| runAsNonRoot | 必须以非 root 用户运行 |
| allowPrivilegeEscalation | 是否允许特权提升 |
| readOnlyRootFilesystem | 根文件系统只读 |
| capabilities | Linux 能力控制 |
| seccompProfile | Seccomp 配置 |

## Pod Security Standards

### 三种安全级别

```
┌─────────────────────────────────────────────────────────────┐
│              Pod Security Standards                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Privileged（特权）：                                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 无限制                                           │   │
│  │  • 适用于系统组件                                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  Baseline（基线）：                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 禁止明显的特权提升                               │   │
│  │  • 适用于大多数应用                                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  Restricted（受限）：                                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 严格限制                                         │   │
│  │  • 适用于安全敏感应用                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Pod Security Admission

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: production
  labels:
    pod-security.kubernetes.io/enforce: restricted
    pod-security.kubernetes.io/enforce-version: latest
    pod-security.kubernetes.io/audit: restricted
    pod-security.kubernetes.io/audit-version: latest
    pod-security.kubernetes.io/warn: restricted
    pod-security.kubernetes.io/warn-version: latest
```

### Restricted 策略示例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: restricted-pod
  namespace: production
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    fsGroup: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    image: myapp:v1
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
    ports:
    - containerPort: 8080
```

## 镜像安全

### 镜像拉取策略

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: image-security
spec:
  containers:
  - name: app
    image: my-registry.example.com/myapp:v1
    imagePullPolicy: Always
  imagePullSecrets:
  - name: registry-secret
```

### 私有镜像仓库

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: registry-secret
type: kubernetes.io/dockerconfigjson
data:
  .dockerconfigjson: <base64-encoded-config>
```

## 网络安全

### NetworkPolicy

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: api-network-policy
spec:
  podSelector:
    matchLabels:
      app: api
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 3306
```

## 最佳实践

### 1. 以非 root 用户运行

```yaml
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
```

### 2. 只读根文件系统

```yaml
securityContext:
  readOnlyRootFilesystem: true
```

### 3. 禁用特权提升

```yaml
securityContext:
  allowPrivilegeEscalation: false
```

### 4. 丢弃所有能力

```yaml
securityContext:
  capabilities:
    drop:
    - ALL
```

### 5. 使用 Seccomp

```yaml
securityContext:
  seccompProfile:
    type: RuntimeDefault
```

## 面试回答

**问题**: Kubernetes 中如何管理容器的安全性？

**回答**: Kubernetes 容器安全管理从多个层面进行：

**Security Context**：配置容器运行时的安全参数。**runAsUser/runAsGroup** 指定运行用户和组；**runAsNonRoot** 强制以非 root 用户运行；**allowPrivilegeEscalation** 禁止特权提升；**readOnlyRootFilesystem** 设置根文件系统只读；**capabilities** 控制 Linux 能力，建议 drop ALL；**seccompProfile** 配置 Seccomp 限制系统调用。

**Pod Security Standards**：定义三种安全级别。**Privileged** 无限制，适用于系统组件；**Baseline** 禁止明显特权提升，适用于大多数应用；**Restricted** 严格限制，适用于安全敏感应用。通过 Namespace 标签配置 Pod Security Admission。

**镜像安全**：使用可信镜像源；配置 imagePullPolicy: Always；使用私有镜像仓库配置 imagePullSecrets；进行镜像安全扫描。

**网络安全**：使用 NetworkPolicy 控制 Pod 间网络访问；配置 Ingress 和 Egress 规则；实现网络隔离。

**最佳实践**：以非 root 用户运行容器；设置只读根文件系统；禁用特权提升；丢弃所有 Linux 能力；使用 Seccomp 限制系统调用；配置资源限制；使用 Pod Security Standards。
