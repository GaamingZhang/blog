# Kubernetes 敏感信息传递

## 引言

在 Kubernetes 中，敏感信息（如密码、密钥、证书等）需要安全地传递给应用。Kubernetes 提供了 Secret 资源来存储和管理敏感信息，同时支持多种安全措施保护敏感数据的传递和存储。

## Secret 基本使用

### 创建 Secret

```bash
kubectl create secret generic db-secret \
  --from-literal=username=admin \
  --from-literal=password='S3cr3t!'

kubectl create secret tls tls-secret \
  --cert=path/to/tls.crt \
  --key=path/to/tls.key
```

### Secret YAML 定义

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
  namespace: production
type: Opaque
data:
  username: YWRtaW4=
  password: UzNjcjN0IQ==
stringData:
  username: admin
  password: S3cr3t!
```

### 在 Pod 中使用 Secret

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-secret
spec:
  containers:
  - name: app
    image: myapp:v1
    env:
    - name: DB_USERNAME
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: username
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password
```

### 挂载 Secret 为文件

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-secret-volume
spec:
  containers:
  - name: app
    image: myapp:v1
    volumeMounts:
    - name: secrets
      mountPath: /etc/secrets
      readOnly: true
  volumes:
  - name: secrets
    secret:
      secretName: db-secret
      defaultMode: 0400
```

## Secret 安全措施

### 1. etcd 加密存储

```yaml
apiVersion: apiserver.config.k8s.io/v1
kind: EncryptionConfiguration
resources:
  - resources:
    - secrets
    providers:
    - aescbc:
        keys:
        - name: key1
          secret: <BASE64_ENCODED_SECRET>
    - identity: {}
```

### 2. RBAC 权限控制

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: secret-reader
  namespace: production
rules:
- apiGroups: [""]
  resources: ["secrets"]
  resourceNames: ["db-secret"]
  verbs: ["get"]
```

### 3. 限制 Secret 访问

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-pod
spec:
  automountServiceAccountToken: false
  containers:
  - name: app
    image: myapp:v1
```

## 外部密钥管理

### External Secrets Operator

```yaml
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: vault-backend
spec:
  provider:
    vault:
      server: "https://vault.example.com"
      path: "secret"
      version: "v2"
      auth:
        kubernetes:
          mountPath: "kubernetes"
          role: "my-role"
---
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: vault-secret
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: vault-backend
    kind: SecretStore
  target:
    name: db-secret
    creationPolicy: Owner
  data:
  - secretKey: password
    remoteRef:
      key: database
      property: password
```

### HashiCorp Vault

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: vault-agent
spec:
  serviceAccountName: app-sa
  containers:
  - name: app
    image: myapp:v1
    volumeMounts:
    - name: secrets
      mountPath: /etc/secrets
  - name: vault-agent
    image: vault:1.13
    args:
    - agent
    - -config=/etc/vault/config.hcl
    volumeMounts:
    - name: vault-config
      mountPath: /etc/vault
    - name: secrets
      mountPath: /etc/secrets
  volumes:
  - name: vault-config
    configMap:
      name: vault-config
  - name: secrets
    emptyDir:
      medium: Memory
```

## Secret 最佳实践

### 1. 使用 immutable Secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: immutable-secret
immutable: true
data:
  username: YWRtaW4=
  password: UzNjcjN0IQ==
```

### 2. 限制文件权限

```yaml
volumes:
- name: secrets
  secret:
    secretName: db-secret
    defaultMode: 0400
```

### 3. 定期轮换密钥

```yaml
metadata:
  annotations:
    rotate/interval: "30d"
    rotate/last-rotated: "2024-01-01"
```

### 4. 使用命名空间隔离

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
  namespace: production
```

## 面试回答

**问题**: Kubernetes 中如何安全地传递敏感信息？

**回答**: Kubernetes 通过 Secret 和多种安全措施安全传递敏感信息：

**Secret 基本使用**：Secret 用于存储敏感信息，支持 Base64 编码存储。可以通过环境变量或挂载文件方式传递给 Pod。环境变量方式简单但可能泄露（进程列表可见）；挂载文件方式更安全，建议设置只读权限和最小文件权限（0400）。

**安全措施**：**etcd 加密**使用 EncryptionConfiguration 配置加密提供者，确保 Secret 在 etcd 中加密存储；**RBAC 控制**通过 Role 和 RoleBinding 限制 Secret 访问权限，遵循最小权限原则；**禁用自动挂载**设置 automountServiceAccountToken: false，避免不必要的 Token 挂载。

**外部密钥管理**：生产环境推荐使用外部密钥管理系统。**HashiCorp Vault** 提供 Secret 注入和动态密钥；**External Secrets Operator** 同步外部密钥到 Kubernetes Secret；**AWS Secrets Manager/Azure Key Vault** 云平台密钥管理服务。

**最佳实践**：使用 immutable Secret 防止意外修改；限制 Secret 文件权限为只读；定期轮换密钥；使用命名空间隔离不同环境的 Secret；审计 Secret 访问日志；避免在 Pod 定义中硬编码敏感信息。
