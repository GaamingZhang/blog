# Kubernetes Pod 探针详解

## 引言

在 Kubernetes 中，确保应用的可靠性是运维的核心任务。Pod 探针（Probe）是 Kubernetes 提供的健康检查机制，通过定期检测容器的健康状态，自动重启不健康的容器或将其从服务负载均衡中移除，从而保证应用的高可用性。

## 探针类型概述

### 三种探针类型

```
┌─────────────────────────────────────────────────────────────┐
│                Kubernetes 探针类型                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  livenessProbe（存活探针）                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 检测容器是否存活                                  │   │
│  │  • 失败时重启容器                                    │   │
│  │  • 防止死锁、僵尸进程                                │   │
│  │  • 不影响 Service 流量                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  readinessProbe（就绪探针）                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 检测容器是否就绪                                  │   │
│  │  • 失败时从 Service 移除                             │   │
│  │  • 控制流量分发                                      │   │
│  │  • 不重启容器                                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  startupProbe（启动探针）                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • 检测容器是否启动完成                              │   │
│  │  • 成功后才开始其他探针                              │   │
│  │  • 适用于慢启动应用                                  │   │
│  │  • 避免被 livenessProbe 误杀                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 探针执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                探针执行流程                                   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  容器启动                                                    │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              startupProbe 执行                       │   │
│  │  • 等待应用启动完成                                  │   │
│  │  • 失败则重启容器                                    │   │
│  │  • 成功后进入下一阶段                                │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │         livenessProbe + readinessProbe              │   │
│  │  • 并行执行                                          │   │
│  │  • livenessProbe 失败 -> 重启容器                    │   │
│  │  • readinessProbe 失败 -> 移出 Service               │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                      │
│       ▼                                                      │
│  容器正常运行                                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## livenessProbe 存活探针

### 基本配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: liveness-demo
spec:
  containers:
  - name: app
    image: nginx:1.21
    livenessProbe:
      httpGet:
        path: /health
        port: 80
      initialDelaySeconds: 15
      periodSeconds: 10
      timeoutSeconds: 5
      failureThreshold: 3
      successThreshold: 1
```

### HTTP GET 探针

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: liveness-http
spec:
  containers:
  - name: app
    image: my-app:v1
    ports:
    - containerPort: 8080
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
        httpHeaders:
        - name: X-Custom-Header
          value: health-check
        scheme: HTTP
      initialDelaySeconds: 30
      periodSeconds: 10
      timeoutSeconds: 5
      failureThreshold: 3
```

### TCP Socket 探针

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: liveness-tcp
spec:
  containers:
  - name: redis
    image: redis:7.0
    ports:
    - containerPort: 6379
    livenessProbe:
      tcpSocket:
        port: 6379
      initialDelaySeconds: 15
      periodSeconds: 10
      timeoutSeconds: 5
      failureThreshold: 3
```

### Exec 探针

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: liveness-exec
spec:
  containers:
  - name: app
    image: my-app:v1
    livenessProbe:
      exec:
        command:
        - /bin/sh
        - -c
        - |
          if [ -f /tmp/healthy ]; then
            exit 0
          else
            exit 1
          fi
      initialDelaySeconds: 5
      periodSeconds: 5
      timeoutSeconds: 1
      failureThreshold: 3
```

  replicas: 3

## startupProbe 启动探针

### 基本配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: startup-demo
spec:
  containers:
  - name: app
    image: my-slow-app:v1
    ports:
    - containerPort: 8080
    startupProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 0
      periodSeconds: 10
      timeoutSeconds: 5
      failureThreshold: 30
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      periodSeconds: 10
      failureThreshold: 3
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      periodSeconds: 5
      failureThreshold: 3
```

### 启动探针适用场景

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: slow-startup-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: slow-app
  template:
    metadata:
      labels:
        app: slow-app
    spec:
      containers:
      - name: app
        image: java-app:v1
        ports:
        - containerPort: 8080
        startupProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 10
          failureThreshold: 30
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 10
          failureThreshold: 3
```

## 探针参数详解

### 参数说明

| 参数 | 说明 | 默认值 |
|-----|------|--------|
| initialDelaySeconds | 容器启动后等待时间 | 0 |
| periodSeconds | 检查间隔 | 10 |
| timeoutSeconds | 超时时间 | 1 |
| failureThreshold | 连续失败次数阈值 | 3 |
| successThreshold | 连续成功次数阈值 | 1 |

### 参数配置示例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: probe-params
spec:
  containers:
  - name: app
    image: my-app:v1
    ports:
    - containerPort: 8080
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10
      timeoutSeconds: 5
      failureThreshold: 3
      successThreshold: 1
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5
      timeoutSeconds: 3
      failureThreshold: 3
      successThreshold: 1
```

## 最佳实践

### 1. 合理设置初始延迟

```yaml
livenessProbe:
  initialDelaySeconds: 30
readinessProbe:
  initialDelaySeconds: 5
```

### 2. 使用 startupProbe 处理慢启动

```yaml
startupProbe:
  periodSeconds: 10
  failureThreshold: 30
```

### 3. 区分 liveness 和 readiness 端点

```yaml
livenessProbe:
  httpGet:
    path: /health/liveness
readinessProbe:
  httpGet:
    path: /health/readiness
```

### 4. 设置合理的超时和阈值

```yaml
livenessProbe:
  timeoutSeconds: 5
  failureThreshold: 3
readinessProbe:
  timeoutSeconds: 3
  failureThreshold: 3
```

## 面试回答

**问题**: 什么是 Pod 的探针？

**回答**: Kubernetes Pod 探针是容器健康检查机制，用于检测容器的健康状态并采取相应措施。

**三种探针类型**：**livenessProbe（存活探针）**检测容器是否存活，失败时重启容器，用于防止死锁、僵尸进程等异常状态；**readinessProbe（就绪探针）**检测容器是否就绪，失败时从 Service Endpoints 移除，控制流量分发，不重启容器；**startupProbe（启动探针）**检测容器是否启动完成，成功后才执行其他探针，适用于慢启动应用，避免被 livenessProbe 误杀。

**三种检测方式**：**httpGet** 发送 HTTP 请求检查响应状态码；**tcpSocket** 检查 TCP 端口是否可连接；**exec** 在容器内执行命令检查返回码。

**关键参数**：initialDelaySeconds（初始延迟）、periodSeconds（检查间隔）、timeoutSeconds（超时时间）、failureThreshold（失败阈值）、successThreshold（成功阈值）。

**执行流程**：容器启动后先执行 startupProbe，成功后并行执行 livenessProbe 和 readinessProbe。livenessProbe 失败重启容器，readinessProbe 失败移出 Service。

**最佳实践**：合理设置初始延迟避免误判；慢启动应用使用 startupProbe；区分 liveness 和 readiness 检查端点；设置合理的超时和阈值；探针端点实现轻量级检查。
