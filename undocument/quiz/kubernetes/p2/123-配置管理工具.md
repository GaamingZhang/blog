# Kubernetes 配置管理工具

## 引言

随着 Kubernetes 集群规模和应用复杂度的增加，手动管理配置文件变得困难且容易出错。配置管理工具通过代码化、自动化和标准化的方式，帮助团队高效管理 Kubernetes 资源配置。本文介绍 Kubernetes 生态中主流的配置管理工具。

## Kustomize

### Kustomize 概述

Kustomize 是 Kubernetes 原生的配置定制工具，无需模板，通过声明式的方式管理配置差异。

### 目录结构

```
app/
├── base/
│   ├── deployment.yaml
│   ├── service.yaml
│   └── kustomization.yaml
└── overlays/
    ├── dev/
    │   ├── kustomization.yaml
    │   └── patch.yaml
    ├── staging/
    │   ├── kustomization.yaml
    │   └── patch.yaml
    └── production/
        ├── kustomization.yaml
        └── patch.yaml
```

### base/kustomization.yaml

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
- deployment.yaml
- service.yaml

commonLabels:
  app: myapp

commonAnnotations:
  version: v1.0.0
```

### overlays/production/kustomization.yaml

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

namespace: production

resources:
- ../../base

patchesStrategicMerge:
- deployment-patch.yaml

configMapGenerator:
- name: app-config
  behavior: merge
  literals:
  - ENV=production
  - LOG_LEVEL=info

replicas:
- name: myapp
  count: 3

images:
- name: myapp
  newTag: v1.0.0
```

### 使用 Kustomize

```bash
kubectl kustomize overlays/production

kubectl apply -k overlays/production

kubectl diff -k overlays/production
```

## ArgoCD

### ArgoCD 概述

ArgoCD 是声明式的 GitOps 持续交付工具，自动同步 Git 仓库中的配置到 Kubernetes 集群。

### ArgoCD 架构

```
┌─────────────────────────────────────────────────────────────┐
│                  ArgoCD 架构                                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    Git Repository                     │  │
│  │  • 存储应用配置                                      │  │
│  │  • 版本控制                                          │  │
│  └──────────────────────────────────────────────────────┘  │
│                          │                                   │
│                          ▼                                   │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    ArgoCD Server                       │  │
│  │  • 监听 Git 变更                                     │  │
│  │  • 比较期望状态与实际状态                            │  │
│  │  • 自动/手动同步                                     │  │
│  └──────────────────────────────────────────────────────┘  │
│                          │                                   │
│                          ▼                                   │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                  Kubernetes Cluster                   │  │
│  │  • 部署应用                                          │  │
│  │  • 维护期望状态                                      │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### ArgoCD Application

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: myapp
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/org/app-config.git
    targetRevision: main
    path: overlays/production
  destination:
    server: https://kubernetes.default.svc
    namespace: production
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
    - CreateNamespace=true
```

### ArgoCD Project

```yaml
apiVersion: argoproj.io/v1alpha1
kind: AppProject
metadata:
  name: production
  namespace: argocd
spec:
  description: Production project
  sourceRepos:
  - 'https://github.com/org/*'
  destinations:
  - namespace: production
    server: https://kubernetes.default.svc
  clusterResourceWhitelist:
  - group: ''
    kind: Namespace
  namespaceResourceBlacklist:
  - group: ''
    kind: ResourceQuota
```

## Flux

### Flux 概述

Flux 是 CNCF 毕业项目，提供 GitOps 自动化能力，支持多租户、多集群管理。

### Flux 组件

```yaml
apiVersion: source.toolkit.fluxcd.io/v1beta2
kind: GitRepository
metadata:
  name: app-config
  namespace: flux-system
spec:
  interval: 1m
  url: https://github.com/org/app-config.git
  ref:
    branch: main
---
apiVersion: kustomize.toolkit.fluxcd.io/v1beta2
kind: Kustomization
metadata:
  name: app
  namespace: flux-system
spec:
  interval: 5m
  path: ./overlays/production
  sourceRef:
    kind: GitRepository
    name: app-config
  prune: true
  validation: client
```

### Flux HelmRelease

```yaml
apiVersion: helm.toolkit.fluxcd.io/v2beta1
kind: HelmRelease
metadata:
  name: myapp
  namespace: production
spec:
  interval: 5m
  chart:
    spec:
      chart: mychart
      version: '1.0.0'
      sourceRef:
        kind: HelmRepository
        name: myrepo
        namespace: flux-system
  values:
    replicaCount: 3
    image:
      tag: v1.0.0
```

## Pulumi

### Pulumi 概述

Pulumi 使用通用编程语言（TypeScript、Python、Go、C#）管理基础设施和 Kubernetes 资源。

### Pulumi 示例

```typescript
import * as k8s from "@pulumi/kubernetes";

const appLabels = { app: "myapp" };

const deployment = new k8s.apps.v1.Deployment("myapp", {
    spec: {
        selector: { matchLabels: appLabels },
        replicas: 3,
        template: {
            metadata: { labels: appLabels },
            spec: {
                containers: [{
                    name: "myapp",
                    image: "myapp:v1.0.0",
                    ports: [{ containerPort: 8080 }],
                }],
            },
        },
    },
});

const service = new k8s.core.v1.Service("myapp", {
    spec: {
        selector: appLabels,
        ports: [{ port: 80, targetPort: 8080 }],
    },
});

export const serviceName = service.metadata.name;
```

## 面试回答

**问题**: 什么是 k8s 的配置管理工具？

**回答**: Kubernetes 配置管理工具用于代码化、自动化管理资源配置，主要分为以下几类：

**模板化工具**：**Helm** 是 Kubernetes 的包管理器，使用 Go Template 将应用打包成 Chart，支持版本管理和依赖管理，适合应用分发和复用；**Kustomize** 是 Kubernetes 原生配置定制工具，无模板语法，通过声明式的方式管理配置差异，使用 base + overlays 结构管理多环境配置。

**GitOps 工具**：**ArgoCD** 是声明式持续交付工具，自动同步 Git 仓库配置到集群，支持多集群管理、可视化界面、回滚等功能；**Flux** 是 CNCF 毕业项目，提供 GitOps 自动化能力，支持 Helm、Kustomize 等多种配置源。

**编程化工具**：**Pulumi** 使用通用编程语言（TypeScript、Python、Go）管理资源，适合复杂配置逻辑；**CDK8s** 使用编程语言定义 Kubernetes 资源，生成标准 YAML。

**IaC 工具**：**Terraform** 用于管理 Kubernetes 集群基础设施；**Ansible** 用于配置自动化。

**选择建议**：简单应用使用 Kustomize；需要打包分发使用 Helm；持续部署使用 ArgoCD 或 Flux；复杂配置逻辑使用 Pulumi；基础设施管理使用 Terraform。

**最佳实践**：采用 GitOps 工作流；环境隔离管理；配置版本控制；自动化同步和回滚。
