# Kubernetes 安全性措施

## 引言

Kubernetes 安全是一个多层次、多维度的问题，涉及集群基础设施、网络通信、应用运行时、数据存储等多个方面。构建安全的 Kubernetes 环境需要从认证、授权、准入控制、网络策略、容器安全等多个层面进行综合防护。

## 认证机制

### 认证方式

| 认证方式 | 说明 |
|---------|------|
| 客户端证书 | 使用 TLS 证书认证 |
| Bearer Token | 使用 Token 认证 |
| 基本认证 | 用户名密码认证 |
| Bootstrap Token | 节点加入集群时使用 |
| OIDC | OpenID Connect 认证 |
| Webhook | 外部认证服务 |

## RBAC 授权

### Role 和 RoleBinding

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: pod-manager
  namespace: production
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch", "create", "update", "delete"]
- apiGroups: [""]
  resources: ["configmaps", "secrets"]
  verbs: ["get", "list"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: pod-manager-binding
  namespace: production
subjects:
- kind: User
  name: dev-user
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-manager
  apiGroup: rbac.authorization.k8s.io
```

## NetworkPolicy 网络策略

### 基本网络策略

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: api-network-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: api
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
    - namespaceSelector:
        matchLabels:
          environment: production
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 3306
  - to:
    - namespaceSelector: {}
      podSelector:
        matchLabels:
          k8s-app: kube-dns
    ports:
    - protocol: UDP
      port: 53
```

### 默认拒绝所有流量

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-all
  namespace: production
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
```

### 允许特定命名空间访问

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy

  name: allow-from-namespace
```

## Security Context 容器安全

### Pod 级别安全上下文

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    runAsGroup: 1000
    fsGroup: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    image: my-app:v1
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
```

### 容器级别安全上下文

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: container-security
spec:
  containers:
  - name: app
    image: my-app:v1
    securityContext:
      runAsNonRoot: true
      runAsUser: 1000
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL
        add:
        - NET_BIND_SERVICE
    volumeMounts:
    - name: tmp
      mountPath: /tmp
  volumes:
  - name: tmp
    emptyDir: {}
```

## 面试回答

**问题**: 什么是 Kubernetes 的安全性措施？

**回答**: Kubernetes 安全性措施是一个多层次的安全防护体系，主要包括以下几个方面：

**集群安全层**：**认证机制**支持客户端证书、Bearer Token、OIDC 等多种认证方式；**RBAC 授权**通过 Role、RoleBinding、ClusterRole、ClusterRoleBinding 实现细粒度权限控制；**准入控制器**通过 ValidatingAdmissionWebhook 和 MutatingAdmissionWebhook 实现请求验证和修改；**审计日志**记录所有 API 操作，便于安全审计。

**网络安全层**：**NetworkPolicy** 实现 Pod 级别的网络隔离，控制入站和出站流量；**Service Mesh**（如 Istio）提供 mTLS 加密通信；**网络插件**（如 Calico）支持网络策略实现。

**容器安全层**：**Security Context** 配置容器运行时的安全参数，如以非 root 用户运行、只读文件系统、禁用特权提升；**Pod Security Standards** 定义了 Privileged、Baseline、Restricted 三种安全级别；**镜像安全**通过镜像扫描、签名验证确保镜像安全。

**数据安全层**：**Secret 加密**使用 EncryptionConfiguration 配置 etcd 加密存储；**RBAC** 限制 Secret 访问权限；**外部密钥管理**集成 HashiCorp Vault 等外部系统。

**最佳实践**：遵循最小权限原则配置 RBAC；使用 NetworkPolicy 实现网络隔离；配置 Security Context 加固容器安全；启用审计日志；定期进行安全扫描和审计。
