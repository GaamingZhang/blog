# Kubernetes 滚动更新详解

## 引言

滚动更新是 Kubernetes Deployment 的默认更新策略，通过逐步替换旧版本的 Pod 来实现应用的无缝升级。滚动更新确保在更新过程中服务始终可用，是生产环境应用更新的首选方式。

## 滚动更新概述

### 滚动更新原理

```
┌─────────────────────────────────────────────────────────────┐
│                  滚动更新原理                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  初始状态：v1 (5 副本)                                       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                            │
│  │v1 │ │v1 │ │v1 │ │v1 │ │v1 │                            │
│  └───┘ └───┘ └───┘ └───┘ └───┘                            │
│                                                              │
│  步骤 1：创建 1 个 v2 Pod (maxSurge=1)                       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                      │
│  │v1 │ │v1 │ │v1 │ │v1 │ │v1 │ │v2 │                      │
│  └───┘ └───┘ └───┘ └───┘ └───┘ └───┘                      │
│                                                              │
│  步骤 2：删除 1 个 v1 Pod (maxUnavailable=1)                 │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                            │
│  │v1 │ │v1 │ │v1 │ │v1 │ │v2 │                            │
│  └───┘ └───┘ └───┘ └───┘ └───┘                            │
│                                                              │
│  继续循环...                                                 │
│                                                              │
│  最终状态：v2 (5 副本)                                       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                            │
│  │v2 │ │v2 │ │v2 │ │v2 │ │v2 │                            │
│  └───┘ └───┘ └───┘ └───┘ └───┘                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 滚动更新参数

| 参数 | 说明 | 默认值 |
|-----|------|--------|
| maxSurge | 最多可以创建的额外 Pod 数量 | 25% |
| maxUnavailable | 最多不可用的 Pod 数量 | 25% |

## 滚动更新配置

### 基本 Deployment 配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 5
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:v1
        ports:
        - containerPort: 8080
```

### 完整滚动更新配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 5
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
  revisionHistoryLimit: 10
  progressDeadlineSeconds: 600
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:v2
        ports:
        - containerPort: 8080
        readinessProbe:
          httpGet:
            path: /ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "500m"
            memory: "512Mi"
```

## 执行滚动更新

### 更新镜像

```bash
kubectl set image deployment/myapp myapp=myapp:v2

kubectl set image deployment/myapp myapp=myapp:v2 --record
```

### 更新配置

```bash
kubectl apply -f deployment.yaml

kubectl edit deployment myapp
```

### 查看更新状态

```bash
kubectl rollout status deployment/myapp

kubectl rollout status deployment/myapp -w

kubectl get pods -l app=myapp -w
```

### 查看更新历史

```bash
kubectl rollout history deployment/myapp

kubectl rollout history deployment/myapp --revision=2
```

## 滚动更新策略

### 保守策略（零停机）

```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1
    maxUnavailable: 0
```

### 激进策略（快速更新）

```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 50%
    maxUnavailable: 50%
```

### 渐进策略

```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1
    maxUnavailable: 1
```

## 滚动更新回滚

### 回滚到上一版本

```bash
kubectl rollout undo deployment/myapp
```

### 回滚到指定版本

```bash
kubectl rollout undo deployment/myapp --to-revision=2
```

### 查看回滚状态

```bash
kubectl rollout status deployment/myapp

kubectl rollout history deployment/myapp
```

## 滚动更新暂停和恢复

### 暂停更新

```bash
kubectl rollout pause deployment/myapp
```

### 恢复更新

```bash
kubectl rollout resume deployment/myapp
```

### 使用场景

```bash
kubectl rollout pause deployment/myapp

kubectl set image deployment/myapp myapp=myapp:v2

kubectl set resources deployment/myapp -c myapp --limits=cpu=500m,memory=512Mi

kubectl rollout resume deployment/myapp
```

## 滚动更新最佳实践

### 1. 配置健康检查

```yaml
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
livenessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 15
  periodSeconds: 10
```

### 2. 设置合理的更新参数

```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1
    maxUnavailable: 0
```

### 3. 使用 --record 记录变更

```bash

```

### 4. 监控更新过程

```bash

kubectl get events --field-selector involvedObject.name=myapp
```

## 面试回答

**问题**: 如何进行滚动更新？

**回答**: Kubernetes 滚动更新是 Deployment 的默认更新策略，通过逐步替换旧版本 Pod 实现无缝升级：

**配置方式**：在 Deployment 的 strategy 字段配置滚动更新参数。maxSurge 控制最多可以创建的额外 Pod 数量，默认 25%；maxUnavailable 控制最多不可用的 Pod 数量，默认 25%。

**执行更新**：使用 `kubectl set image deployment/myapp container=image:v2` 更新镜像；或修改 YAML 文件后执行 `kubectl apply -f deployment.yaml`。

**更新流程**：创建新版本 Pod -> 等待新 Pod 就绪 -> 删除旧版本 Pod -> 循环直到所有 Pod 更新完成。readinessProbe 确保新 Pod 就绪后才接收流量。

**查看状态**：`kubectl rollout status deployment/myapp` 查看更新状态；`kubectl rollout history deployment/myapp` 查看更新历史。

**回滚操作**：`kubectl rollout undo deployment/myapp` 回滚到上一版本；`kubectl rollout undo deployment/myapp --to-revision=N` 回滚到指定版本。

**最佳实践**：配置 readinessProbe 确保新 Pod 就绪；设置 maxUnavailable=0 实现零停机更新；使用 --record 记录变更原因；监控更新过程；准备回滚方案。
