# Kubernetes 跨集群部署和管理

## 引言

随着业务规模扩大，单一 Kubernetes 集群可能无法满足高可用、容灾、多地域部署等需求。跨集群管理成为企业级 Kubernetes 运维的重要能力。通过跨集群部署，可以实现业务的多活、容灾、就近访问等目标。

## Kubernetes Federation

### KubeFed 架构

Kubernetes Federation（KubeFed）是官方提供的跨集群管理方案。

```yaml
apiVersion: core.kubefed.io/v1beta1
kind: KubeFedCluster
metadata:
  name: cluster-beijing
  namespace: kube-federation-system
spec:
  apiEndpoint: https://cluster-beijing.example.com
  caBundle: <ca-bundle>
  tlsClientConfig:
    cert: <client-cert>
    key: <client-key>
---
apiVersion: core.kubefed.io/v1beta1
kind: KubeFedCluster
metadata:
  name: cluster-shanghai
  namespace: kube-federation-system
spec:
  apiEndpoint: https://cluster-shanghai.example.com
  caBundle: <ca-bundle>
  tlsClientConfig:
    cert: <client-cert>
    key: <client-key>
```

### 联邦资源分发

```yaml
apiVersion: types.kubefed.io/v1beta1
kind: FederatedDeployment
metadata:
  name: federated-app
  namespace: production
spec:
  placement:
    clusters:
    - name: cluster-beijing
    - name: cluster-shanghai
  overrides:
  - clusterName: cluster-beijing
    clusterOverrides:
    - path: spec.replicas
      value: 3
  - clusterName: cluster-shanghai
    clusterOverrides:
    - path: spec.replicas
      value: 2
  template:
    metadata:
      labels:
        app: federated-app
    spec:
      replicas: 3
      selector:
        matchLabels:
          app: federated-app
      template:
        metadata:
          labels:
            app: federated-app
        spec:
          containers:
          - name: app
            image: my-app:v1
            ports:
            - containerPort: 8080
```

### 联邦 Service

```yaml
apiVersion: types.kubefed.io/v1beta1

      - port: 80
```

```yaml

  name: multi-cluster-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/org/app-config.git
```

### 1. 统一认证授权

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: cross-cluster-admin
rules:
- apiGroups: [""]
  resources: ["*"]
  verbs: ["*"]

  name: cross-cluster-admin-binding

- kind: User
  name: admin@example.com
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: ClusterRole
  name: cross-cluster-admin
  apiGroup: rbac.authorization.k8s.io
```

## 面试回答

**问题**: 如何进行跨集群部署和管理？

**回答**: Kubernetes 跨集群部署和管理主要通过以下方案实现：

**联邦管理方案**：使用 Kubernetes Federation（KubeFed）进行跨集群资源分发和管理。KubeFed 提供统一的控制平面，可以将 Deployment、Service、ConfigMap 等资源分发到多个集群，支持按集群定制配置（如不同集群不同副本数）。

**多集群服务发现**：通过 ServiceExport 和 ServiceImport 实现跨集群服务发现，使用 CoreDNS 解析 clusterset.local 域名，实现跨集群服务访问。

**应用分发工具**：使用 ArgoCD、Flux 等 GitOps 工具进行多集群应用部署，支持从 Git 仓库同步配置到多个集群；使用 Kustomize 管理不同集群的差异化配置。

**网络互联**：使用 Submariner 或 Cilium ClusterMesh 实现集群间网络连通，使 Pod 可以直接跨集群通信，无需 NAT。

**监控与备份**：配置联邦 Prometheus 采集多集群监控数据，使用 Velero 进行跨集群备份和恢复。

**典型应用场景**：多地域部署实现就近访问、容灾备份实现故障切换、混合云部署兼顾安全与弹性、弹性扩展应对流量高峰。

最佳实践：统一认证授权、统一策略管理、合理规划资源配额、配置跨集群网络互联、建立完善的监控告警体系。
