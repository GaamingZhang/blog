# Kubernetes 集群备份和恢复

## 引言

Kubernetes 集群备份是保障业务连续性的重要措施。通过定期备份集群资源和持久化数据，可以在发生故障时快速恢复。本文介绍 Kubernetes 集群备份和恢复的方案和最佳实践。

## etcd 备份

### 手动备份 etcd

```bash
ETCDCTL_API=3 etcdctl snapshot save snapshot.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key
```

### 恢复 etcd

```bash
ETCDCTL_API=3 etcdctl snapshot restore snapshot.db \
  --data-dir=/var/lib/etcd-restore

# 修改 etcd 配置指向新数据目录
# 重启 etcd
```

### 定期备份脚本

```bash
#!/bin/bash
BACKUP_DIR="/backup/etcd"
DATE=$(date +%Y%m%d_%H%M%S)

ETCDCTL_API=3 etcdctl snapshot save ${BACKUP_DIR}/etcd-${DATE}.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key

# 保留最近 7 天的备份
find ${BACKUP_DIR} -name "etcd-*.db" -mtime +7 -delete
```

## Velero 备份

### Velero 安装

```bash
velero install \
  --provider aws \
  --plugins velero/velero-plugin-for-aws:v1.7.0 \
  --bucket velero-backup \
  --secret-file ./credentials-velero \
  --backup-location-config region=us-east-1 \
  --use-volume-snapshots=true \
  --snapshot-location-config region=us-east-1
```

### 创建备份

```bash
velero backup create full-backup --include-namespaces default,production

velero backup create daily-backup --schedule="0 2 * * *"

velero backup create app-backup --selector app=myapp
```

### Velero 备份配置

```yaml
apiVersion: velero.io/v1
kind: Backup
metadata:
  name: scheduled-backup
  namespace: velero
spec:
  includedNamespaces:
  - production
  - staging
  excludedResources:
  - events
  - pods
  storageLocation: default
  volumeSnapshotLocations:
  - default
  ttl: 720h
  hooks:
    resources:
    - name: pre-backup-hook
      includedNamespaces:
      - production
      labelSelector:
        matchLabels:
          app: database
      pre:
      - exec:
          container: mysql
          command:
          - /bin/bash
          - -c
          - mysqldump --all-databases > /backup/dump.sql
          onError: Continue
```

### 恢复备份

```bash
velero restore create --from-backup full-backup

velero restore create --from-backup daily-backup --namespace-mappings production:production-restore
```

### Velero 恢复配置

```yaml
apiVersion: velero.io/v1
kind: Restore
metadata:
  name: restore-from-backup
  namespace: velero
spec:
  backupName: full-backup
  includedNamespaces:
  - production
  restorePVs: true
  preserveNodePorts: true
```

## 资源导出备份

### 导出所有资源

```bash
kubectl get all -A -o yaml > all-resources.yaml

kubectl get configmaps,secrets -A -o yaml > configs.yaml
```

### 导出特定命名空间

```bash
kubectl get all -n production -o yaml > production.yaml

kubectl get configmaps,secrets -n production -o yaml > production-configs.yaml
```

### 使用脚本定期备份

```bash
#!/bin/bash
BACKUP_DIR="/backup/resources"
DATE=$(date +%Y%m%d_%H%M%S)
NAMESPACE=${1:-production}

mkdir -p ${BACKUP_DIR}/${DATE}

kubectl get all -n ${NAMESPACE} -o yaml > ${BACKUP_DIR}/${DATE}/resources.yaml
kubectl get configmaps -n ${NAMESPACE} -o yaml > ${BACKUP_DIR}/${DATE}/configmaps.yaml
kubectl get secrets -n ${NAMESPACE} -o yaml > ${BACKUP_DIR}/${DATE}/secrets.yaml
kubectl get pvc -n ${NAMESPACE} -o yaml > ${BACKUP_DIR}/${DATE}/pvc.yaml
```

## PV 数据备份

### 使用 Volume Snapshot

```yaml
apiVersion: snapshot.storage.k8s.io/v1
kind: VolumeSnapshot
metadata:
  name: pvc-snapshot
  namespace: production
spec:
  volumeSnapshotClassName: csi-snapclass
  source:
    persistentVolumeClaimName: data-pvc
```

### 恢复 Volume Snapshot

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: restored-pvc
  namespace: production
spec:
  accessModes:
  - ReadWriteOnce
  storageClassName: standard
  dataSource:
    name: pvc-snapshot
    kind: VolumeSnapshot
    apiGroup: snapshot.storage.k8s.io
  resources:
    requests:
      storage: 10Gi
```

## 面试回答

**问题**: 如何备份和恢复 Kubernetes 集群？

**回答**: Kubernetes 集群备份和恢复包括 etcd 备份、资源备份、PV 数据备份等：

**etcd 备份**：etcd 存储 Kubernetes 所有状态数据。使用 `etcdctl snapshot save` 命令备份 etcd 数据。恢复时使用 `etcdctl snapshot restore` 恢复数据，然后重启 etcd 服务。建议定期备份（如每天），并保留多个版本。

**Velero 备份**：Velero 是 Kubernetes 备份的标准工具。支持备份集群资源和 PV 数据。支持定时备份、选择性备份、跨集群恢复。配置云存储作为备份后端，支持 S3、GCS、Azure Blob 等。

**资源导出备份**：使用 `kubectl get -o yaml` 导出资源定义。适用于快速备份和版本控制。可以备份到 Git 仓库实现 GitOps。

**PV 数据备份**：使用 Volume Snapshot 创建 PV 快照。使用 Velero 的 Volume Snapshot 功能自动备份 PV。对于数据库，建议先执行逻辑备份（如 mysqldump）。

**灾难恢复**：评估故障范围 -> 选择恢复策略 -> 执行恢复 -> 验证结果。建议制定详细的灾难恢复计划，定期演练恢复流程。

**最佳实践**：定期自动备份；异地备份存储；定期测试恢复；监控备份状态；保留多个备份版本；文档化恢复流程。
