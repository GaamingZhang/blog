# Kubernetes 水平和垂直扩展

## 引言

在 Kubernetes 中，应用扩展是应对负载变化的关键能力。扩展分为水平扩展和垂直扩展两种方式。水平扩展通过增加 Pod 副本数量来提升处理能力，垂直扩展通过增加单个 Pod 的资源配置来提升性能。理解两种扩展方式的适用场景和配置方法，对于构建弹性、高效的容器化应用至关重要。

## 扩展方式概述

### 水平扩展 vs 垂直扩展

```
┌─────────────────────────────────────────────────────────────┐
│          水平扩展 vs 垂直扩展                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  水平扩展：                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  原理：增加 Pod 副本数量                             │   │
│  │                                                      │   │
│  │  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐                    │   │
│  │  │Pod│ │Pod│ │Pod│ │Pod│ │Pod│  (多个副本)        │   │
│  │  └───┘ └───┘ └───┘ └───┘ └───┘                    │   │
│  │                                                      │   │
│  │  优点：高可用性、负载均衡、无单点故障                │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  垂直扩展：                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  原理：增加单个 Pod 的资源配置                       │   │
│  │                                                      │   │
│  │  ┌─────────────────────────────────────────────┐    │   │
│  │  │                  Pod                         │    │   │
│  │  │        CPU: 2 cores, Memory: 4Gi            │    │   │
│  │  └─────────────────────────────────────────────┘    │   │
│  │                                                      │   │
│  │  优点：适合有状态应用、资源密集型应用                │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 扩展方式选择

| 场景 | 推荐方式 | 原因 |
|-----|---------|------|
| 无状态 Web 服务 | 水平扩展 | 易于扩展，高可用 |
| 数据库 | 垂直扩展 | 有状态，难以水平扩展 |
| 批处理任务 | 水平扩展 | 并行处理效率高 |
| 内存密集型应用 | 垂直扩展 | 需要大内存 |
| 微服务 | 水平扩展 | 弹性伸缩，高可用 |

## 水平扩展（HPA）

### HPA 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                  HPA 工作原理                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Metrics Server -> HPA Controller -> Deployment             │
│                                                              │
│  计算公式：                                                  │
│  期望副本数 = ceil(当前副本数 × (当前指标值 / 目标指标值))  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 基于 CPU 的 HPA

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: cpu-hpa
  namespace: production
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

### HPA 行为配置

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: hpa-with-behavior
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300
      policies:
      - type: Percent
        value: 10
        periodSeconds: 60
    scaleUp:
      stabilizationWindowSeconds: 0
      policies:
      - type: Percent
        value: 100
        periodSeconds: 15
```

## 垂直扩展（VPA）

### VPA 基本配置

```yaml
apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: my-app-vpa
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  updatePolicy:
    updateMode: Auto
  resourcePolicy:
    containerPolicies:
    - containerName: app
      minAllowed:
        cpu: 100m
        memory: 128Mi
      maxAllowed:
        cpu: 2000m
        memory: 2Gi
```

### VPA 更新模式

| 模式 | 说明 |
|-----|------|
| Auto | 自动应用推荐值，会重启 Pod |
| Recreate | 自动应用推荐值，会重启 Pod |
| Initial | 仅在 Pod 创建时应用推荐值 |
| Off | 仅生成推荐，不应用 |

### VPA 推荐模式

```yaml
apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: vpa-recommend
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  updatePolicy:
    updateMode: Off
```

## 面试回答

**问题**: 什么是 Kubernetes 中的水平和垂直扩展？

**回答**: Kubernetes 扩展分为水平扩展和垂直扩展两种方式：

**水平扩展（HPA）**：通过增加 Pod 副本数量来提升处理能力。HPA（HorizontalPodAutoscaler）根据 CPU、内存或自定义指标自动调整副本数。计算公式：期望副本数 = ceil(当前副本数 × (当前指标值 / 目标指标值))。适用于无状态应用，提供高可用性和负载均衡。

**垂直扩展（VPA）**：通过增加单个 Pod 的资源配置（CPU、内存）来提升性能。VPA（VerticalPodAutoscaler）监控 Pod 资源使用，自动调整 requests 和 limits。适用于有状态应用和资源密集型应用，但调整时需要重启 Pod。

**选择建议**：无状态 Web 服务使用水平扩展；数据库等有状态应用使用垂直扩展；微服务使用水平扩展；内存密集型应用使用垂直扩展。HPA 和 VPA 可以结合使用，但避免同时基于相同指标，推荐 HPA 使用自定义指标，VPA 调整资源配置。

**最佳实践**：合理设置 minReplicas 和 maxReplicas；配置 scaleDown 稳定窗口避免频繁波动；VPA 设置资源范围限制；监控扩展效果并持续优化。
