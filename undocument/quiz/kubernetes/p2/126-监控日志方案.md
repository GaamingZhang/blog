# Kubernetes 监控和日志记录解决方案

## 引言

监控和日志是 Kubernetes 运维的核心能力，通过监控可以实时了解集群和应用状态，通过日志可以排查问题和审计操作。构建完善的监控和日志系统，对于保障集群稳定运行至关重要。

## 监控解决方案

### Prometheus 监控栈

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: prometheus
  namespace: monitoring
spec:
  replicas: 1
  selector:
    matchLabels:
      app: prometheus
  template:
    spec:
      containers:
      - name: prometheus
        image: prom/prometheus:v2.45.0
        args:
        - "--config.file=/etc/prometheus/prometheus.yml"
        - "--storage.tsdb.path=/prometheus"
        - "--storage.tsdb.retention.time=15d"
        ports:
        - containerPort: 9090
        volumeMounts:
        - name: config
          mountPath: /etc/prometheus
        - name: storage
          mountPath: /prometheus
```

### Prometheus 配置

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-config
data:
  prometheus.yml: |
    global:
      scrape_interval: 15s
      evaluation_interval: 15s

    scrape_configs:
    - job_name: 'kubernetes-apiservers'
      kubernetes_sd_configs:
      - role: endpoints
      scheme: https
      tls_config:
        ca_file: /var/run/secrets/kubernetes.io/serviceaccount/ca.crt
      bearer_token_file: /var/run/secrets/kubernetes.io/serviceaccount/token

    - job_name: 'kubernetes-nodes'
      kubernetes_sd_configs:
      - role: node
      relabel_configs:
      - source_labels: [__address__]
        regex: (.+):(.+)
        target_label: __address__
        replacement: ${1}:9100

    - job_name: 'kubernetes-pods'
      kubernetes_sd_configs:
      - role: pod
      relabel_configs:
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true
```

### 告警规则

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-rules
data:
  alert-rules.yml: |
    groups:
    - name: node-alerts
      rules:
      - alert: NodeHighCPU
        expr: 100 - (avg by(instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "节点 CPU 使用率过高"

      - alert: NodeHighMemory
        expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100 > 85
        for: 5m
        labels:
          severity: warning
```

### Fluentd DaemonSet

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  namespace: logging
spec:
  selector:
    matchLabels:
      app: fluentd
  template:
    metadata:
      labels:
        app: fluentd
    spec:
      serviceAccountName: fluentd
      tolerations:
      - key: node-role.kubernetes.io/master
        effect: NoSchedule
      containers:
      - name: fluentd
        image: fluent/fluentd-kubernetes-daemonset:v1-debian-elasticsearch
        env:
        - name: FLUENT_ELASTICSEARCH_HOST
          value: "elasticsearch.logging.svc.cluster.local"
        - name: FLUENT_ELASTICSEARCH_PORT
          value: "9200"
        - name: FLUENT_ELASTICSEARCH_SCHEME
          value: "http"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers
```

## 日志类型

### 标准输出日志

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: stdout-logs
spec:
  containers:
  - name: app
    image: nginx:1.21
    command: ["/bin/sh", "-c"]
    args:
    - while true; do echo "$(date) - Hello from stdout"; sleep 1; done
```

### 文件日志

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: file-logs
spec:
  containers:
  - name: app
    image: nginx:1.21
    volumeMounts:
    - name: logs
      mountPath: /var/log/app
  volumes:
  - name: logs
    emptyDir: {}
```

## 最佳实践

### 1. 监控指标选择

```
• CPU 使用率
• 内存使用率
• 磁盘 I/O
• 网络流量
• 请求延迟
• 错误率
```

### 2. 日志格式规范

```json
{
  "timestamp": "2024-01-01T10:00:00Z",
  "level": "INFO",
  "message": "Request processed",
  "service": "myapp",
  "trace_id": "abc123",
  "user_id": "user1"

```

```yaml
severity: critical  # 严重告警，需要立即处理

```

## 面试回答

**问题**: 什么是 Kubernetes 的监控和日志记录解决方案？

**回答**: Kubernetes 监控和日志记录是运维的核心能力：

**监控解决方案**：采用 Prometheus + Grafana + Alertmanager 技术栈。**Prometheus** 是核心监控组件，负责数据采集、存储和告警规则评估。通过 Node Exporter 采集节点指标，kube-state-metrics 采集 Kubernetes 对象状态，cAdvisor 采集容器指标。**Grafana** 提供可视化 Dashboard，展示监控数据。**Alertmanager** 处理告警通知，支持邮件、Slack、钉钉等多种通知渠道。

**日志解决方案**：采用 EFK（Elasticsearch + Fluentd/Fluent Bit + Kibana）技术栈。**Fluentd/Fluent Bit** 以 DaemonSet 方式部署在每个节点，采集容器标准输出和文件日志。**Elasticsearch** 存储和索引日志数据，支持全文检索和聚合分析。**Kibana** 提供日志搜索、可视化和分析功能。

**日志类型**：**标准输出日志**通过容器运行时采集，最简单；**文件日志**需要挂载存储卷，适用于需要持久化的日志。

**最佳实践**：监控指标选择 CPU、内存、磁盘、网络、延迟、错误率等关键指标；日志采用 JSON 格式，包含时间戳、级别、消息、服务名、追踪 ID 等字段；告警分级为 critical、warning、info；配置合理的数据保留策略；建立完善的告警响应流程。
