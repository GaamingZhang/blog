# Kubernetes 概述

## 引言

Kubernetes（简称 K8s）是当今最流行的容器编排平台，它彻底改变了应用程序的部署、扩展和管理方式。作为云原生技术的核心组件，Kubernetes 提供了一个强大的平台，用于自动化部署、扩展和管理容器化应用程序。

本文将深入介绍 Kubernetes 的核心概念、主要目标、架构设计以及在现代软件开发中的重要地位。

## 什么是 Kubernetes

### 定义

Kubernetes 是一个开源的容器编排平台，最初由 Google 开发，现在由云原生计算基金会（CNCF）维护。它的名字来源于希腊语，意为"舵手"或"飞行员"。

### 与 Borg 的关系

Kubernetes 的设计灵感来源于 Google 内部的 Borg 系统：

| 特性 | Borg | Kubernetes |
|-----|------|-----------|
| **开发时间** | 2003年 | 2014年 |
| **使用范围** | Google 内部 | 开源社区 |
| **设计理念** | 大规模作业调度 | 容器编排平台 |
| **API 设计** | 内部 API | RESTful API |

### 设计原则

1. **声明式 API**：用户声明期望状态，系统负责实现
2. **控制器模式**：通过控制器持续调谐实际状态与期望状态
3. **微服务架构**：组件松耦合，易于扩展
4. **可移植性**：支持多种云平台和基础设施
5. **可扩展性**：支持自定义资源和控制器

## Kubernetes 核心概念

### Pod

Pod 是 Kubernetes 最小的部署单元，包含一个或多个容器：

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80
```

### Service

Service 提供稳定的服务访问入口：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
```

### Deployment

Deployment 管理 Pod 的副本和更新：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-app:v1
```

### ConfigMap 和 Secret

```yaml
# ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  database_url: "mysql://db:3306/mydb"
---
# Secret
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque
stringData:
  password: mypassword
```

## Kubernetes 适用场景

### 典型应用场景

| 场景 | 说明 |
|-----|------|
| **微服务架构** | 服务发现、负载均衡、配置管理 |
| **持续集成/部署** | 滚动更新、回滚、蓝绿部署 |
| **大数据处理** | 批处理任务、定时任务 |
| **机器学习** | GPU 调度、模型训练、推理服务 |
| **混合云部署** | 多云管理、跨云调度 |

### 不适用场景

- 简单的单体应用
- 不需要弹性伸缩的应用
- 对延迟极其敏感的应用
- 需要直接访问硬件的应用

## 面试回答

**问题**: 什么是 Kubernetes？它的主要目标是什么？

**回答**: Kubernetes（简称 K8s）是一个开源的容器编排平台，最初由 Google 开发，现在由云原生计算基金会（CNCF）维护。它的名字来源于希腊语，意为"舵手"。Kubernetes 的设计灵感来源于 Google 内部的 Borg 系统，于 2014 年开源，2015 年发布 1.0 版本，现已成为容器编排的事实标准。

Kubernetes 的主要目标包括：**自动化部署和回滚**，支持声明式配置管理和滚动更新；**服务发现和负载均衡**，内置服务发现机制和自动负载均衡；**存储编排**，自动挂载存储系统，支持多种存储后端；**自动装箱**，根据资源需求调度容器，优化资源利用率；**自我修复**，自动重启失败容器，替换和重新调度 Pod；**密钥和配置管理**，管理敏感信息和配置热更新；**水平扩缩容**，支持手动和自动扩缩容。

Kubernetes 的核心设计原则包括：声明式 API 让用户声明期望状态；控制器模式持续调谐实际状态与期望状态；微服务架构使组件松耦合易于扩展；可移植性支持多种云平台；可扩展性支持自定义资源。

Kubernetes 适用于微服务架构、持续集成/部署、大数据处理、机器学习、混合云部署等场景，是云原生技术栈的核心组件。
