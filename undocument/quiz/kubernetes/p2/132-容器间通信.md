# Kubernetes 容器间通信

## 引言

在 Kubernetes 中，容器间通信是微服务架构的基础。理解容器间通信的机制，包括同一 Pod 内容器通信、同节点 Pod 间通信、跨节点 Pod 间通信，对于设计和排查容器网络问题至关重要。

## 容器间通信类型

### 通信类型概述

```
┌─────────────────────────────────────────────────────────────┐
│              容器间通信类型                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 同一 Pod 内容器通信                                      │
│     • 共享网络命名空间                                       │
│     • 通过 localhost 通信                                   │
│                                                              │
│  2. 同节点 Pod 间通信                                        │
│     • 通过 veth pair 和网桥                                 │
│     • 直接路由                                              │
│                                                              │
│  3. 跨节点 Pod 间通信                                        │
│     • 通过 CNI 插件                                         │
│     • Overlay 网络或路由                                    │
│                                                              │
│  4. Pod 与 Service 通信                                      │
│     • 通过 ClusterIP                                        │
│     • kube-proxy 负载均衡                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 同一 Pod 内容器通信

### 共享网络命名空间

```
┌─────────────────────────────────────────────────────────────┐
│              同一 Pod 内容器通信                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                      Pod                              │  │
│  │                                                       │  │
│  │   ┌─────────────┐     ┌─────────────┐               │  │
│  │   │ Container A │     │ Container B │               │  │
│  │   │ Port: 8080  │     │ Port: 9090  │               │  │
│  │   └──────┬──────┘     └──────┬──────┘               │  │
│  │          │                   │                       │  │
│  │          │    共享网络命名空间    │                   │  │
│  │          │         │         │                       │  │
│  │          ▼         ▼         ▼                       │  │
│  │   ┌─────────────────────────────────────┐            │  │
│  │   │         Pause Container             │            │  │
│  │   │         (网络命名空间持有者)          │            │  │
│  │   │         IP: 10.244.1.2              │            │  │
│  │   └─────────────────────────────────────┘            │  │
│  │                                                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
│  Container A 访问 Container B: localhost:9090               │
│  Container B 访问 Container A: localhost:8080               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 配置示例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: multi-container
spec:
  containers:
  - name: app
    image: myapp:v1
    ports:
    - containerPort: 8080
    volumeMounts:
    - name: shared-logs
      mountPath: /var/log/app
  - name: log-collector
    image: fluentd:v1
    ports:
    - containerPort: 24224
    volumeMounts:
    - name: shared-logs
      mountPath: /var/log/app
      readOnly: true
  volumes:
  - name: shared-logs
    emptyDir: {}
```

### 通信方式

```bash
# Container A 访问 Container B
curl http://localhost:24224

# Container B 访问 Container A
curl http://localhost:8080
```

## 同节点 Pod 间通信

### 通信架构

```
┌─────────────────────────────────────────────────────────────┐
│              同节点 Pod 间通信                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                      Node                             │  │
│  │                                                       │  │
│  │   ┌─────────┐        ┌─────────┐                     │  │
│  │   │  Pod A  │        │  Pod B  │                     │  │
│  │   │10.244.1.2│       │10.244.1.3│                    │  │
│  │   └────┬────┘        └────┬────┘                     │  │
│  │        │                  │                           │  │
│  │        ▼                  ▼                           │  │
│  │   ┌─────────────────────────────────┐                │  │
│  │   │         cni0 (bridge)           │                │  │
│  │   │      10.244.1.1/24              │                │  │
│  │   └─────────────────────────────────┘                │  │
│  │                                                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
│  Pod A -> cni0 -> Pod B                                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 通信流程

```
1. Pod A 发送数据包到 Pod B (10.244.1.3)
2. 数据包通过 veth pair 到达 cni0 网桥
3. 网桥根据 MAC 地址转发到 Pod B 的 veth pair
4. Pod B 接收数据包
```

## 跨节点 Pod 间通信

### 通信架构

```
┌─────────────────────────────────────────────────────────────┐
│              跨节点 Pod 间通信                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────────┐              ┌──────────────────┐    │
│  │     Node 1       │              │     Node 2       │    │
│  │                  │              │                  │    │
│  │  ┌─────────┐     │              │     ┌─────────┐  │    │
│  │  │  Pod A  │     │              │     │  Pod B  │  │    │
│  │  │10.244.1.2│    │              │     │10.244.2.2│ │    │
│  │  └────┬────┘     │              │     └────┬────┘  │    │
│  │       │          │              │          │       │    │
│  │  ┌────┴────┐     │              │     ┌────┴────┐  │    │
│  │  │   cni0  │     │              │     │   cni0  │  │    │
│  │  └────┬────┘     │              │     └────┬────┘  │    │
│  │       │          │              │          │       │    │
│  │  ┌────┴────┐     │              │     ┌────┴────┐  │    │
│  │  │  flannel│     │              │     │  flannel│  │    │
│  │  └────┬────┘     │              │     └────┬────┘  │    │
│  └───────┼──────────┘              └──────────┼───────┘    │
│          │                                     │            │
│          └──────────────► Overlay ◄────────────┘            │
│                          Network (VXLAN)                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 通信流程

```
1. Pod A 发送数据包到 Pod B (10.244.2.2)
2. 数据包到达 Node 1 的 cni0 网桥
3. flannel 判断目标 IP 属于其他节点
4. 数据包封装为 VXLAN 包，发送到 Node 2
5. Node 2 的 flannel 解封装
6. 数据包通过 cni0 网桥转发到 Pod B
```

```
│  └──────────────────────────────────────────────────────┘  │
│                          │                                   │

│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Service 配置

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: myapp
  ports:
  - port: 80
    targetPort: 8080
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:v1
        ports:
        - containerPort: 8080
```

## 网络策略控制通信

### NetworkPolicy 配置

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: api-network-policy
spec:
  podSelector:
    matchLabels:
      app: api
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
    ports:
    - protocol: TCP
      port: 8080
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 3306
```

## 通信调试

### 测试 Pod 间通信

```bash
kubectl exec -it <pod-name> -- ping <target-pod-ip>

kubectl exec -it <pod-name> -- curl http://<service-name>:80

kubectl exec -it <pod-name> -- nc -zv <target-ip> <port>
```

### 查看网络配置

```bash
kubectl get pods -o wide

kubectl describe pod <pod-name>

kubectl exec -it <pod-name> -- ip addr

kubectl exec -it <pod-name> -- route -n
```

## 面试回答

**问题**: 怎样进行容器间通信？

**回答**: Kubernetes 容器间通信分为四种类型：

**同一 Pod 内容器通信**：Pod 内所有容器共享网络命名空间，通过 localhost 直接通信。Pause 容器持有网络命名空间，其他容器加入该命名空间。容器 A 访问容器 B 使用 `localhost:端口`。

**同节点 Pod 间通信**：通过 veth pair 和网桥（如 cni0）实现。每个 Pod 有一对 veth pair，一端在 Pod 内，一端连接到网桥。Pod A 发送数据包到网桥，网桥根据 MAC 地址转发到 Pod B。

**跨节点 Pod 间通信**：通过 CNI 插件（如 Flannel、Calico）实现。Flannel 使用 VXLAN 封装数据包，在节点间建立 Overlay 网络。Pod A 发送数据包到目标 IP，CNI 插件判断目标在其他节点，封装后发送到目标节点解封装。

**Pod 与 Service 通信**：通过 Service ClusterIP 和 kube-proxy 实现。kube-proxy 维护 iptables/IPVS 规则，将 Service IP 转发到后端 Pod IP。支持轮询等负载均衡算法。

**网络策略控制**：使用 NetworkPolicy 控制 Pod 间的网络访问，实现入站和出站流量控制。

**调试方法**：使用 ping、curl、nc 等命令测试连通性；使用 ip addr、route -n 查看网络配置；使用 kubectl describe 查看网络状态。
