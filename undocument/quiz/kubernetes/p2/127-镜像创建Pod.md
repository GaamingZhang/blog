# 从镜像创建 Pod

## 引言

Pod 是 Kubernetes 中最小的部署单元，包含一个或多个容器。从镜像创建 Pod 是 Kubernetes 最基本的操作。理解 Pod 的创建过程和配置方法，是使用 Kubernetes 的基础。

## 基本创建方法

### 命令行创建

```bash
kubectl run nginx --image=nginx:1.21 --port=80

kubectl run nginx --image=nginx:1.21 --replicas=3

kubectl run nginx --image=nginx:1.21 --dry-run=client -o yaml > pod.yaml
```

### YAML 文件创建

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80
```

```bash
kubectl apply -f pod.yaml

kubectl create -f pod.yaml
```

## Pod 配置详解

### 基本配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
  namespace: default
  labels:
    app: myapp
    environment: production
  annotations:
    description: "My application pod"
spec:
  containers:
  - name: myapp
    image: myapp:v1
    ports:
    - containerPort: 8080
      protocol: TCP
```

### 资源配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: resource-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    resources:
      requests:
        cpu: "100m"
        memory: "128Mi"
      limits:
        cpu: "500m"
        memory: "512Mi"
```

### 环境变量配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: env-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    env:
    - name: APP_ENV
      value: "production"
    - name: DB_HOST
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: db_host
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: app-secret
          key: password
```

### 存储卷配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: volume-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    volumeMounts:
    - name: config
      mountPath: /etc/app
      readOnly: true
    - name: data
      mountPath: /var/data
  volumes:
  - name: config
    configMap:
      name: app-config
  - name: data
    persistentVolumeClaim:
      claimName: data-pvc
```

### 健康检查配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: health-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5
```

## 多容器 Pod

### Sidecar 模式

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: sidecar-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    ports:
    - containerPort: 8080
    volumeMounts:
    - name: logs
      mountPath: /var/log/app
  - name: log-collector
    image: fluent/fluentd:v1.14
    volumeMounts:
    - name: logs
      mountPath: /var/log/app
      readOnly: true
  volumes:
  - name: logs
    emptyDir: {}
```

## 镜像拉取策略

### imagePullPolicy 配置

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: image-pull-pod
spec:
  containers:
  - name: app
    image: myapp:v1
    imagePullPolicy: Always
```

### 拉取策略说明

| 策略 | 说明 |
|-----|------|
| Always | 每次都拉取镜像 |
| IfNotPresent | 本地不存在时拉取（默认） |
| Never | 从不拉取，只使用本地镜像 |

### 私有镜像仓库

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: private-image-pod
spec:
  imagePullSecrets:
  - name: registry-secret
  containers:
  - name: app
    image: my-registry.example.com/myapp:v1
    imagePullPolicy: Always
```

## Pod 状态

### Pod 生命周期阶段

| 阶段 | 说明 |
|-----|------|
| Pending | Pod 已创建，等待调度 |
| Running | Pod 已调度，容器运行中 |
| Succeeded | Pod 中所有容器成功终止 |
| Failed | Pod 中所有容器终止，至少一个失败 |
| Unknown | 无法获取 Pod 状态 |

### 查看 Pod 状态

```bash
kubectl get pods

kubectl get pod <pod-name> -o wide

kubectl describe pod <pod-name>

kubectl logs <pod-name>

kubectl logs <pod-name> -c <container-name>
```

## 最佳实践

### 1. 使用 Deployment 管理 Pod

```yaml

metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
```

## 面试回答

**问题**: 怎样从一个镜像创建一个 Pod？

**回答**: 从镜像创建 Pod 有两种主要方式：

**命令行方式**：使用 `kubectl run` 命令快速创建 Pod。例如 `kubectl run nginx --image=nginx:1.21 --port=80`。可以使用 `--dry-run=client -o yaml` 生成 YAML 模板。

**YAML 文件方式**：编写 Pod 的 YAML 配置文件，使用 `kubectl apply -f pod.yaml` 创建。YAML 文件包含 apiVersion（v1）、kind（Pod）、metadata（名称、标签、注解）、spec（容器配置）等部分。

**Pod 创建流程**：用户提交请求 -> API Server 认证授权并存储到 etcd -> Scheduler 调度 Pod 到合适节点 -> Kubelet 监听到分配的 Pod -> 拉取镜像 -> 创建并启动容器。

**关键配置**：**镜像配置**指定 image 和 imagePullPolicy；**资源配置**设置 requests 和 limits；**环境变量**通过 env 或 envFrom 配置；**存储卷**通过 volumeMounts 和 volumes 配置；**健康检查**配置 livenessProbe 和 readinessProbe。

**镜像拉取策略**：Always（每次都拉取）、IfNotPresent（本地不存在时拉取，默认）、Never（从不拉取）。私有镜像仓库需要配置 imagePullSecrets。

**最佳实践**：生产环境使用 Deployment 管理 Pod 而不是直接创建 Pod；配置资源限制；配置健康检查；使用 ConfigMap 和 Secret 管理配置和敏感信息。
