# Kubernetes Pod 创建/删除时组件关系变化详解

## 引言

在 Kubernetes 集群中，Service、Endpoint 和 kube-proxy 三个组件协同工作，实现了服务发现和负载均衡功能。当 Pod 被创建或删除时，这三个组件之间会发生一系列复杂的交互和状态变化。

理解这些组件在 Pod 生命周期中的变化过程，对于排查服务发现问题、优化应用部署和深入理解 Kubernetes 网络模型至关重要。本文将深入剖析 Pod 创建和删除过程中，Service、Endpoint 和 kube-proxy 三者之间的关系变化。

## 组件关系总览

### 三者核心关系

├─────────────────────────────────────────────────────────────┤
│                                                              │

│  │                   Service                            │   │
│  │  - 定义服务抽象                                      │   │

│                         │                                   │

│  │  - 维护后端 Pod IP 列表                              │   │

│  │  - 提供健康检查状态                                   │   │

### 组件职责

| 组件 | 类型 | 职责 | 控制器 |
|-----|------|------|--------|
| **Service** | API 对象 | 定义服务抽象 | Service Controller |
| **Endpoints** | API 对象 | 维护后端 Pod 列表 | Endpoints Controller |
| **kube-proxy** | 节点组件 | 实现负载均衡 | kube-proxy |

## 组件详细变化

### Service 变化

Service 在 Pod 创建/删除过程中通常保持不变：

```yaml
# Service 定义保持不变
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
  clusterIP: 10.96.0.100  # 保持不变
```

- addresses:

  - port: 8080

## 面试回答

**问题**: Pod 创建/删除时，Service、Endpoint、kube-proxy 三者的关系变化是怎样的？

**回答**: Pod 创建和删除时，Service、Endpoint、kube-proxy 三者会发生一系列协作变化。

**Pod 创建时**：首先，Pod 被创建并调度到节点，kubelet 启动容器并分配 IP 地址。当 Pod 的 readinessProbe 通过后，Endpoints Controller 检测到 Pod 就绪且标签匹配 Service 的 selector，将 Pod IP 添加到 Endpoints 的 addresses 列表中。kube-proxy 监听到 Endpoints 变化后，更新节点上的 iptables/IPVS 规则，将新 Pod 加入负载均衡池。此时流量开始路由到新 Pod。

**Pod 删除时**：首先，API Server 标记 Pod 为删除状态，设置 deletionTimestamp。Endpoints Controller 检测到 Pod 标记删除，立即从 Endpoints 中移除该 Pod IP。kube-proxy 监听到 Endpoints 变化，更新 iptables/IPVS 规则，从负载均衡池移除该 Pod。此时新流量不再路由到该 Pod，但正在处理的请求会继续。然后 kubelet 执行优雅终止流程：执行 preStop 钩子、发送 SIGTERM 信号、等待 terminationGracePeriodSeconds。最后容器停止，Pod 对象从 etcd 删除。

**关键点**：Endpoints 的更新先于容器停止，确保流量不会路由到正在终止的容器，实现平滑的服务下线。这个机制配合 readinessProbe 和 preStop 钩子，可以实现零停机部署。
