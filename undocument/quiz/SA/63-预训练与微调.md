# 预训练与微调

## 问题:什么是预训练和微调?它们在大模型开发中的作用是什么?

预训练和微调是大模型开发的核心范式。预训练通过海量无标注数据学习通用语言知识,微调则在特定任务数据上适配模型能力。这一范式使大模型能够从通用能力转化为专业应用,大幅降低了AI应用的开发门槛。

## 预训练-微调范式

│                                                                 │

│  ┌──────────────────────────────────────────────────────────┐  │
│  │                                                         │  │

│  │  ┌─────────────┐      ┌──────────┐      ┌──────────┐   │  │
│  │  │ 预训练模型  │ ──▶  │ 任务数据 │ ──▶  │ 专业模型 │   │  │

│  │  └─────────────┘      └──────────┘      └──────────┘   │  │

│  │  · 训练成本显著下降                                     │  │
│  │  · 利用预训练的通用知识                                 │  │
│  │                                                         │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 微调方法详解

```
┌─────────────────────────────────────────────────────────────────┐
│                    微调方法分类                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  1. 全量微调 (Full Fine-tuning)         │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  方法: 更新模型所有参数                                  │   │
│  │                                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  预训练模型 (所有参数)                       │       │   │
│  │  │         ↓                                    │       │   │
│  │  │    [全部更新]                                │       │   │
│  │  │         ↓                                    │       │   │
│  │  │  微调模型 (所有参数改变)                     │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  优点: 效果最好,充分利用任务数据                        │   │
│  │  缺点: 计算成本高,存储成本高,容易过拟合                 │   │
│  │  存储: 每个任务需要保存完整模型                         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  2. 参数高效微调 (PEFT)                  │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  2.1 LoRA (Low-Rank Adaptation)                        │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  原始权重 W (d×d, 冻结)                      │       │   │
│  │  │         +                                    │       │   │
│  │  │  低秩分解 W' = BA                           │       │   │
│  │  │         B (d×r) × A (r×d)                   │       │   │
│  │  │         其中 r << d (如r=8, d=4096)         │       │   │
│  │  │                                             │       │   │
│  │  │  输出 = Wx + BAx                            │       │   │
│  │  │                                             │       │   │
│  │  │  参数量: 2×d×r (相比d²大幅降低)             │       │   │
│  │  │  例如: 2×4096×8 = 65536 << 4096²            │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  2.2 Prefix Tuning                                      │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  在输入前添加可学习的prefix tokens          │       │   │
│  │  │                                             │       │   │
│  │  │  原始输入: [x1, x2, x3, ...]                │       │   │
│  │  │         ↓                                    │       │   │
│  │  │  添加prefix: [p1, p2, ..., pk, x1, x2, ...] │       │   │
│  │  │              └─────┘                        │       │   │
│  │  │            可学习参数                        │       │   │
│  │  │                                             │       │   │
│  │  │  只训练prefix参数,冻结原模型                │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  2.3 Adapter                                            │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  在Transformer层中插入小型网络              │       │   │
│  │  │                                             │       │   │
│  │  │  ┌──────────┐                              │       │   │
│  │  │  │Attention │                              │       │   │
│  │  │  └────┬─────┘                              │       │   │
│  │  │       ↓                                     │       │   │
│  │  │  ┌──────────┐  ← Adapter 1                 │       │   │
│  │  │  │ d→r→d    │  (参数少)                    │       │   │
│  │  │  └────┬─────┘                              │       │   │
│  │  │       ↓                                     │       │   │
│  │  │  ┌──────────┐                              │       │   │
│  │  │  │   FFN    │                              │       │   │
│  │  │  └────┬─────┘                              │       │   │
│  │  │       ↓                                     │       │   │
│  │  │  ┌──────────┐  ← Adapter 2                 │       │   │
│  │  │  │ d→r→d    │                              │       │   │
│  │  │  └──────────┘                              │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  3. 指令微调 (Instruction Tuning)       │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  目的: 让模型学会遵循指令                                │   │
│  │                                                         │   │
│  │  数据格式:                                               │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ {                                           │       │   │
│  │  │   "instruction": "将以下句子翻译成英文",    │       │   │
│  │  │   "input": "今天天气很好",                  │       │   │
│  │  │   "output": "The weather is nice today."   │       │   │
│  │  │ }                                           │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  数据来源:                                               │   │
│  │  · 人工标注的高质量指令数据                             │   │
│  │  · Self-Instruct (模型生成+人工筛选)                   │   │
│  │  · 开源指令数据集 (Alpaca, Dolly等)                    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  4. 对齐微调 (Alignment Tuning)         │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  4.1 RLHF (Reinforcement Learning from Human Feedback) │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  步骤1: 监督微调 (SFT)                       │       │   │
│  │  │  用高质量对话数据微调模型                    │       │   │
│  │  │                                             │       │   │
│  │  │  步骤2: 训练奖励模型 (RM)                    │       │   │
│  │  │  人工对模型输出排序 → 训练打分模型           │       │   │
│  │  │                                             │       │   │
│  │  │  步骤3: 强化学习优化 (PPO)                   │       │   │
│  │  │  用奖励模型指导策略模型优化                  │       │   │
│  │  │                                             │       │   │
│  │  │  ┌───────┐    ┌───────┐    ┌───────┐      │       │   │
│  │  │  │  SFT  │───▶│  RM   │───▶│  PPO  │      │       │   │
│  │  │  └───────┘    └───────┘    └───────┘      │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  4.2 DPO (Direct Preference Optimization)              │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  直接用偏好数据优化,无需训练奖励模型        │       │   │
│  │  │  更简单高效,效果接近RLHF                    │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 微调流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    微调实施流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  完整微调流程                            │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  步骤1: 任务定义                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ · 明确任务类型 (分类/生成/抽取等)           │       │   │
│  │  │ · 定义输入输出格式                          │       │   │
│  │  │ · 确定评估指标                              │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  步骤2: 数据准备                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ · 收集任务相关数据                          │       │   │
│  │  │ · 数据清洗和标注                            │       │   │
│  │  │ · 划分训练/验证/测试集                      │       │   │
│  │  │ · 数据增强 (可选)                           │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  步骤3: 模型选择                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ · 选择基座模型 (GPT/LLaMA/Qwen等)           │       │   │
│  │  │ · 确定模型规模 (7B/13B/70B等)               │       │   │
│  │  │ · 选择微调方法 (全量/LoRA/Adapter等)        │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  步骤4: 训练配置                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ · 学习率: 通常1e-5到5e-5                    │       │   │
│  │  │ · Batch Size: 根据显存调整                  │       │   │
│  │  │ · Epochs: 通常2-5轮                         │       │   │
│  │  │ · 优化器: AdamW                             │       │   │
│  │  │ · 学习率调度: 线性衰减                      │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  步骤5: 训练执行                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ · 监控训练loss和验证指标                    │       │   │

## 常见问题

### Q1: 预训练和微调的数据量差异有多大?

预训练和微调的数据量差异巨大。预训练通常使用TB级甚至PB级的无标注文本数据,例如GPT-3使用了45TB文本,约3000亿tokens。而微调通常只需要MB到GB级的标注数据,例如指令微调可能只需要几万到几十万条高质量指令数据。这种差异的原因是:预训练需要学习通用的语言知识和世界知识,需要海量数据覆盖各种领域和场景;微调只需要适配特定任务,在预训练知识基础上进行定向优化,因此数据需求大幅降低。LoRA等参数高效微调方法甚至可以在几千条数据上取得不错的效果。

### Q2: 如何选择合适的微调方法?

选择微调方法需要考虑多个因素:第一,计算资源,如果有充足的GPU资源(如多张A100),可以考虑全量微调获得最佳效果;如果资源有限(单张消费级GPU),LoRA是最佳选择。第二,数据规模,数据量大(>10万条)可以考虑全量微调,数据量小(<1万条)建议使用LoRA防止过拟合。第三,任务需求,通用对话应用需要指令微调+RLHF对齐,垂直领域应用可以用领域数据微调,多任务场景适合LoRA或Adapter。第四,部署成本,如果需要部署多个任务模型,LoRA只需存储小的适配器参数,成本更低。建议从LoRA开始实验,效果好再考虑全量微调。

### Q3: LoRA为什么有效?

LoRA的有效性基于两个假设:第一,模型适应特定任务时,权重更新的内在维度很低,即可以用低秩矩阵近似;第二,预训练模型已经学到了丰富的特征,微调只需要在这些特征上进行小幅调整。LoRA通过低秩分解将参数量降低几个数量级(如从0.1%到1%),但效果接近全量微调。其优势还包括:不增加推理延迟(可以合并到原权重),易于切换任务(只需替换小的LoRA权重),减少存储成本。实践表明,LoRA在秩为8-64时就能达到接近全量微调的效果,是性价比最高的微调方法之一。

### Q4: 指令微调和RLHF有什么区别?

指令微调和RLHF是两个不同阶段的微调方法。指令微调(SFT)是监督学习过程,使用人工编写的指令-回答对训练模型,目的是让模型学会理解和遵循指令,能够按照要求完成任务。RLHF则是在指令微调基础上的进一步优化,包括三个步骤:首先用高质量对话数据做SFT,然后训练奖励模型学习人类偏好,最后用强化学习(PPO)优化模型使其输出更符合人类期望。RLHF主要解决对齐问题,让模型更安全、更有用、更诚实。简单来说,指令微调让模型"会做事",RLHF让模型"做得更好、更符合人类价值观"。实践中,通常先做指令微调,再做RLHF。

### Q5: 微调时如何防止过拟合?

防止微调过拟合的方法包括:第一,数据层面,确保数据质量和多样性,避免重复数据,可以使用数据增强技术;第二,训练层面,使用较小的学习率(如1e-5到5e-5),减少训练轮数(通常2-5轮),采用早停策略,当验证集性能不再提升时停止训练;第三,正则化,使用Dropout、权重衰减等技术,LoRA等参数高效微调方法天然具有正则化效果;第四,数据量,确保有足够的训练数据,数据量太少容易过拟合,建议至少几千条高质量数据;第五,验证集,划分充足的验证集监控过拟合,及时调整训练策略。实践中,LoRA配合早停策略是防止过拟合的有效组合。

## 小结

预训练和微调是大模型开发的核心范式。预训练通过海量无标注数据学习通用语言知识,需要TB级数据和巨大算力投入。微调在预训练基础上适配特定任务,只需少量标注数据即可实现专业化。微调方法包括全量微调、参数高效微调(LoRA、Prefix Tuning、Adapter)、指令微调和RLHF对齐微调。选择微调方法需要权衡计算资源、数据规模、任务需求和部署成本。LoRA以其低成本、高效率成为最流行的微调方法。指令微调让模型学会遵循指令,RLHF实现人类偏好对齐。微调时需要注意防止过拟合,通过数据质量控制、训练策略优化和验证集监控来确保模型泛化能力。

## 面试回答总结

预训练和微调是大模型开发的两阶段范式。预训练通过海量无标注数据(如GPT-3用45TB文本)学习通用语言知识,训练目标是语言模型或掩码语言模型,需要巨大算力投入(如数千万美元)。微调在预训练基础上用少量标注数据适配特定任务,大幅降低开发成本。微调方法包括:全量微调(更新所有参数,效果最好但成本高)、参数高效微调如LoRA(只训练低秩适配器,参数量仅0.1-1%,效果接近全量微调)、指令微调(让模型学会遵循指令)、RLHF(通过人类反馈强化学习实现安全对齐)。选择方法需考虑资源、数据量、任务需求。LoRA因其低成本高效益成为主流选择。微调时需注意防止过拟合,通过早停、正则化、数据质量控制等手段确保模型泛化能力。这一范式使大模型能够从通用能力转化为专业应用,降低了AI应用的门槛。
