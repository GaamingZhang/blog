# 大模型微调方法

## 问题：大模型的微调方法有哪些？如何选择合适的微调策略？

大模型微调是指在预训练模型基础上，使用特定领域或任务的数据进行进一步训练，使模型适应特定场景需求的技术。选择合适的微调策略需要平衡效果、成本、数据量等多个因素，是大模型应用落地的关键技术。

## 参数高效微调(PEFT)

```
┌─────────────────────────────────────────────────────────────────┐
│                    参数高效微调方法                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  1. LoRA(低秩适应)                       │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心原理:                                               │   │
│  │  ┌──────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  原始权重矩阵W (冻结,不更新)                 │       │   │
│  │  │      W                                      │       │   │
│  │  │     d×d                                     │       │   │
│  │  │                                             │       │   │
│  │  │  低秩分解:                                   │       │   │
│  │  │      ΔW = B × A                             │       │   │
│  │  │           d×r  r×d    (r << d)              │       │   │
│  │  │                                             │       │   │
│  │  │  最终输出:                                   │       │   │
│  │  │      h = Wx + ΔWx = Wx + BAx               │       │   │
│  │  │                                             │       │   │
│  │  │  只训练A和B矩阵,参数量大幅减少              │       │   │
│  │  │                                             │       │   │
│  │  └──────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  参数配置:                                               │   │
│  │  ┌──────────────────────────────────────────────┐       │   │
│  │  │ 参数              说明              典型值    │       │   │
│  │  ├──────────────────────────────────────────────┤       │   │
│  │  │ rank(r)          低秩维度          8-64      │       │   │
│  │  │ alpha            缩放因子          16-32     │       │   │
│  │  │ target_modules   应用层            q,v,k,o   │       │   │
│  │  │ dropout          正则化            0.05      │       │   │
│  │  └──────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  优势:                                                   │   │
│  │  · 参数量减少90%以上                                    │   │
│  │  · 显存占用大幅降低                                     │   │
│  │  · 效果接近全参数微调                                   │   │
│  │  · 可训练多个LoRA适配不同任务                           │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  2. QLoRA(量化LoRA)                      │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心技术:                                               │   │
│  │  ┌──────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  1. 4-bit量化:                               │       │   │
│  │  │     将模型权重量化为4-bit整数                │       │   │
│  │  │     显存占用降低75%                          │       │   │
│  │  │                                             │       │   │
│  │  │  2. 双重量化:                                │       │   │
│  │  │     对量化参数再次量化                       │       │   │
│  │  │     进一步减少显存                           │       │   │
│  │  │                                             │       │   │
│  │  │  3. 分页优化器:                              │       │   │
│  │  │     处理显存峰值                             │       │   │
│  │  │     避免OOM                                  │       │   │
│  │  │                                             │       │   │
│  │  └──────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  效果对比:                                               │   │
│  │  ┌──────────────────────────────────────────────┐       │   │
│  │  │ 模型规模    全参数    LoRA    QLoRA          │       │   │
│  │  ├──────────────────────────────────────────────┤       │   │

│  │  │  │                                     │   │       │   │
│  │  │  │  Self-Attention ──▶ Adapter ──▶    │   │       │   │
│  │  │  │                        │            │   │       │   │

│  │  │  │  FFN ──▶ Adapter ──▶ Output        │   │       │   │
│  │  │  │                                     │   │       │   │

│  │  │                                             │       │   │
```

## 常见问题

### Q1: 微调和RAG如何选择？

微调和RAG的选择取决于具体需求：微调适用于需要改变模型行为、学习特定任务格式、提升特定领域能力的场景，优势是推理更快、效果更深入，但需要大量标注数据和计算资源，知识更新需要重新训练。RAG适用于知识密集型场景、需要实时更新知识、访问私有数据的情况，优势是成本低、更新快、可解释性强，但推理时需要检索步骤。实践中，两者可以结合：先用RAG解决知识问题，再用微调优化特定任务表现。建议优先考虑RAG，当RAG无法满足需求时再考虑微调。

### Q2: 如何避免灾难性遗忘？

灾难性遗忘是指微调后模型忘记了预训练阶段学习到的通用能力。避免方法包括：第一，混合训练，在微调数据中混入一定比例的原始预训练数据，保持模型通用能力；第二，降低学习率，使用较小的学习率（如1e-5），减少对原始参数的破坏；第三，参数高效微调，使用LoRA等方法，只更新少量参数，保留大部分原始知识；第四，多任务学习，同时训练多个任务，保持模型多样性；第五，知识蒸馏，使用原始模型作为教师模型，约束微调后的输出。实践中，建议定期评估模型的通用能力，及时发现遗忘问题。

### Q3: LoRA的rank参数如何选择？

LoRA的rank参数影响模型的表达能力和参数量。选择原则：第一，任务复杂度，简单任务（如分类）rank=8即可，复杂任务（如对话）rank=32-64；第二，数据量，数据量大可以适当增大rank，数据量小则减小rank避免过拟合；第三，实验验证，从rank=8开始，逐步增加，观察效果提升是否明显；第四，计算资源，rank越大参数越多，训练和推理成本越高。实践中，大多数场景rank=16-32就能达到接近全参数微调的效果，建议从rank=16开始实验。同时注意alpha参数通常设为rank的2倍。

### Q4: 如何评估微调效果？

评估微调效果需要多维度进行：第一，自动评估指标，使用BLEU、ROUGE评估生成质量，使用准确率评估分类任务，使用Perplexity评估语言模型能力；第二，人工评估，评估回答的准确性、相关性、流畅性、安全性，通常使用1-5分评分；第三，对比评估，与原始模型对比，与基线方法对比，与商业模型对比；第四，业务指标，在实际应用中评估用户满意度、问题解决率等；第五，能力保持，评估模型是否保持了通用能力，避免灾难性遗忘。建议建立评估数据集，定期评估并记录结果，持续优化。

### Q5: 微调需要多少数据？

微调所需数据量取决于多个因素：第一，任务复杂度，简单任务（如分类、格式转换）1K-5K样本可能就够了，复杂任务（如对话、推理）需要10K-100K样本；第二，微调方法，全参数微调需要更多数据，LoRA等参数高效方法可以用较少数据；第三，数据质量，高质量数据可以用更少的量达到更好效果；第四，模型规模，大模型通常需要更多数据才能充分微调；第五，效果要求，追求更好效果需要更多数据。实践中，建议从少量数据（1K-5K）开始实验，根据效果逐步增加数据量，同时注重数据质量而非单纯追求数量。

## 小结

大模型微调方法分为全参数微调（SFT、RLHF、DPO）和参数高效微调（LoRA、QLoRA、Adapter、Prefix Tuning）。全参数微调效果最佳但成本高，适用于需要深度改变模型行为的场景。参数高效微调大幅降低成本，效果接近全参数微调，适合资源受限场景。LoRA是最常用的PEFT方法，通过低秩分解只训练0.1%-1%的参数。QLoRA进一步通过量化将显存需求降低到原来的1/4。选择微调策略需要考虑数据量、算力资源、任务需求、效果要求等因素。最佳实践包括：准备高质量数据、合理设置超参数、监控训练过程、多维度评估效果。微调与RAG可以结合使用，分别解决能力改变和知识增强问题。

## 面试回答总结

大模型微调方法主要分为全参数微调和参数高效微调(PEFT)两大类。全参数微调包括：SFT(监督微调),使用标注数据更新所有参数,适合改变模型行为风格;RLHF(人类反馈强化学习),通过奖励模型和PPO优化使输出符合人类偏好,适合安全对齐;DPO(直接偏好优化),简化RLHF流程,直接用偏好数据优化,效果接近RLHF但更简单。参数高效微调包括：LoRA(低秩适应),只训练低秩分解矩阵,参数量减少90%以上,效果接近全参数微调;QLoRA,在LoRA基础上使用4-bit量化,显存需求降低到原来的1/4;Adapter,在Transformer层间插入小型网络;Prefix Tuning,在输入前添加可训练前缀向量。选择策略：有大量标注数据且需要改变模型行为选全参数SFT;显存有限选QLoRA;多任务切换选LoRA;需要对齐人类价值观选RLHF或DPO;快速实验选LoRA/QLoRA。关键参数：learning_rate建议1e-5到5e-5,LoRA的rank建议8-64,alpha设为rank的2倍。避免灾难性遗忘的方法包括混合训练原始数据、降低学习率、使用PEFT方法。
