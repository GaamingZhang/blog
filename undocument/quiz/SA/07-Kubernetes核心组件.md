# Kubernetes核心组件详解

## Kubernetes架构概述

Kubernetes（简称K8s）是一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用。它采用Master-Worker架构，由控制平面（Control Plane）和工作节点（Worker Node）组成。

```
Kubernetes整体架构：

┌─────────────────────────────────────────────────────────────────────────┐
│                           Kubernetes Cluster                             │
│                                                                         │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │                      Control Plane (Master)                        │  │
│  │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │  │
│  │  │ API Server  │ │    etcd     │ │  Scheduler  │ │ Controller  │ │  │
│  │  │             │ │             │ │             │ │   Manager   │ │  │
│  │  │  集群入口    │ │  数据存储   │ │   调度器    │ │   控制器    │ │  │
│  │  └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘ │  │
│  │                                                                   │  │
│  │  ┌─────────────────────────────────────────────────────────────┐ │  │
│  │  │                      Cloud Controller Manager               │ │  │
│  │  │                      (云厂商控制器，可选)                     │ │  │
│  │  └─────────────────────────────────────────────────────────────┘ │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                    │                                    │
│                    ┌───────────────┼───────────────┐                   │
│                    ↓               ↓               ↓                   │
│  ┌─────────────────────┐ ┌─────────────────────┐ ┌─────────────────────┐│
│  │    Worker Node 1    │ │    Worker Node 2    │ │    Worker Node 3    ││
│  │  ┌───────────────┐  │ │  ┌───────────────┐  │ │  ┌───────────────┐  ││
│  │  │    Kubelet    │  │ │  │    Kubelet    │  │ │  │    Kubelet    │  ││
│  │  └───────────────┘  │ │  └───────────────┘  │ │  └───────────────┘  ││
│  │  ┌───────────────┐  │ │  ┌───────────────┐  │ │  ┌───────────────┐  ││
│  │  │ kube-proxy    │  │ │  │ kube-proxy    │  │ │  │ kube-proxy    │  ││
│  │  └───────────────┘  │ │  └───────────────┘  │ │  └───────────────┘  ││

│  │  └───────────────┘  │ │  └───────────────┘  │ │  └───────────────┘  ││
│  │  ┌─────┐ ┌─────┐   │ │  ┌─────┐ ┌─────┐   │ │  ┌─────┐ ┌─────┐   ││
│  │  │ Pod │ │ Pod │   │ │  │ Pod │ │ Pod │   │ │  │ Pod │ │ Pod │   ││
```

## 控制平面组件

### 1. API Server

API Server是Kubernetes控制平面的前端，是集群的统一入口。

```
API Server核心职责：

┌─────────────────────────────────────────────────────────────────────────┐
│                           kube-apiserver                                 │
│                                                                         │
│  核心功能：                                                               │
│  ├── RESTful API：提供集群管理的REST接口                                 │
│  ├── 认证（Authentication）：验证用户身份                                │
│  ├── 授权（Authorization）：验证用户权限                                 │
│  ├── 准入控制（Admission Control）：验证和修改请求                       │
│  └── 数据验证：验证资源对象的合法性                                      │
│                                                                         │
│  请求流程：                                                               │
│                                                                         │
│  kubectl get pods                                                       │
│       │                                                                 │
│       ↓                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                      API Server                                  │   │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐              │   │
│  │  │ 认证    │→│ 授权    │→│准入控制 │→│ 验证    │              │   │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘              │   │
│  │                            ↓                                    │   │
│  │                      ┌─────────┐                               │   │
│  │                      │  etcd   │                               │   │
│  │                      └─────────┘                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  关键参数：                                                               │
│  ├── --advertise-address：API Server对外地址                            │
│  ├── --secure-port：HTTPS端口（默认6443）                               │
│  ├── --etcd-servers：etcd集群地址                                       │
│  └── --authorization-mode：授权模式                                     │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2. etcd

etcd是Kubernetes的后端存储，保存集群的所有数据。

```
etcd架构：

┌─────────────────────────────────────────────────────────────────────────┐
│                            etcd集群                                      │
│                                                                         │
│  高可用部署（3或5节点）：                                                  │
│                                                                         │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐              │
│  │   etcd-1    │ ←→  │   etcd-2    │ ←→  │   etcd-3    │              │
│  │  (Leader)   │     │ (Follower)  │     │ (Follower)  │              │
│  └─────────────┘     └─────────────┘     └─────────────┘              │
│        │                   │                   │                       │
│        └───────────────────┴───────────────────┘                       │
│                     Raft一致性协议                                       │
│                                                                         │
│  存储内容：                                                               │
│  ├── 节点信息（Nodes）                                                   │
│  ├── Pod信息                                                             │
│  ├── Service信息                                                         │
│  ├── ConfigMap和Secret                                                  │
│  ├── RBAC规则                                                           │
│  └── 集群配置                                                            │
│                                                                         │
│  数据结构（键值对）：                                                      │
│  /registry/pods/default/my-pod                                          │
│  /registry/services/default/my-service                                  │
│  /registry/configmaps/default/my-config                                 │
│                                                                         │
│  备份命令：                                                               │
│  etcdctl snapshot save snapshot.db                                      │
│  etcdctl snapshot restore snapshot.db                                   │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3. Scheduler

Scheduler负责将Pod调度到合适的节点上运行。

```
调度流程：

┌─────────────────────────────────────────────────────────────────────────┐
│                           kube-scheduler                                 │
│                                                                         │
│  调度流程：                                                               │
│                                                                         │
│  Pod创建请求                                                             │
│       │                                                                 │
│       ↓                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                    预选（Filtering）                             │   │
│  │  过滤不满足条件的节点：                                           │   │
│  │  ├── 节点资源是否充足                                            │   │
│  │  ├── 节点是否有污点                                              │   │
│  │  ├── Pod是否有节点亲和性要求                                      │   │
│  │  └── 节点是否满足PVC要求                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│       │                                                                 │
│       ↓                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                    优选（Scoring）                               │   │
│  │  对候选节点打分：                                                 │   │
│  │  ├── 资源利用率                                                  │   │
│  │  ├── 亲和性/反亲和性                                             │   │
│  │  ├── 拓扑分布                                                    │   │
│  │  └── 自定义打分                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│       │                                                                 │
│       ↓                                                                 │
│  选择得分最高的节点                                                       │
│       │                                                                 │
│       ↓                                                                 │
│  绑定Pod到节点                                                           │
│                                                                         │
│  调度策略示例：                                                           │
│  nodeSelector:                                                          │
│    disktype: ssd                                                        │
│  affinity:                                                              │
│    nodeAffinity:                                                        │
│      requiredDuringSchedulingIgnoredDuringExecution:                    │
│        nodeSelectorTerms:                                               │
│        - matchExpressions:                                              │
│          - key: kubernetes.io/arch                                      │
│            operator: In                                                 │
│            values: [amd64]                                              │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4. Controller Manager

Controller Manager运行各种控制器，监控集群状态并进行调谐。

```
控制器类型：

┌─────────────────────────────────────────────────────────────────────────┐
│                        kube-controller-manager                           │
│                                                                         │
│  核心控制器：                                                             │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ Node Controller                                                  │   │
│  │ ├── 监控节点健康状态                                              │   │
│  │ ├── 处理节点不可用情况                                            │   │
│  │ └── 更新节点状态                                                  │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ Replication Controller                                           │   │
│  │ ├── 维护Pod副本数量                                               │   │
│  │ └── 确保期望副本数                                                │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ Deployment Controller                                            │   │
│  │ ├── 管理Deployment                                                │   │
│  │ ├── 滚动更新                                                      │   │
│  │ └── 回滚                                                          │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ Service Controller                                               │   │
│  │ ├── 管理Service                                                   │   │
│  │ └── 负载均衡器管理                                                │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ Endpoint Controller                                              │   │
│  │ ├── 维护Service与Pod的映射                                        │   │
│  │ └── 更新Endpoints                                                 │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ ServiceAccount Controller                                        │   │
│  │ └── 管理ServiceAccount                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  控制器工作模式：                                                         │
│                                                                         │
│  while true:                                                            │
│    current_state = get_current_state()                                  │
│    desired_state = get_desired_state()                                  │
│    if current_state != desired_state:                                   │
│        reconcile()  # 调谐                                              │
│    sleep(period)                                                        │
└─────────────────────────────────────────────────────────────────────────┘
```

## 工作节点组件

### 1. Kubelet

Kubelet是每个节点上的代理，负责维护Pod的生命周期。

```
Kubelet核心职责：

┌─────────────────────────────────────────────────────────────────────────┐
│                            kubelet                                       │
│                                                                         │
│  主要功能：                                                               │
│  ├── Pod生命周期管理                                                     │
│  ├── 容器健康检查                                                        │
│  ├── 节点状态上报                                                        │
│  ├── 资源使用监控                                                        │
│  └── 卷管理                                                              │
│                                                                         │
│  工作流程：                                                               │
│                                                                         │
│  API Server ──→ 监听Pod事件 ──→ kubelet                                 │
│                                      │                                  │
│                                      ↓                                  │
│                              ┌─────────────────┐                        │
│                              │ Pod创建/更新    │                        │
│                              └────────┬────────┘                        │
│                                       │                                 │
│                                       ↓                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                      Pod工作流                                    │   │
│  │  1. 创建Pod目录                                                   │   │
│  │  2. 挂载卷                                                        │   │
│  │  3. 拉取镜像                                                      │   │
│  │  4. 创建容器                                                      │   │
│  │  5. 启动容器                                                      │   │
│  │  6. 执行PostStart钩子                                             │   │
│  │  7. 健康检查                                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  关键配置：                                                               │
│  ├── --node-ip：节点IP地址                                              │
│  ├── --pod-infra-container-image：基础容器镜像                          │
│  ├── --cgroup-driver：cgroup驱动                                       │
│  └── --container-runtime：容器运行时                                    │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2. kube-proxy

kube-proxy负责维护节点上的网络规则，实现Service的负载均衡。

```
kube-proxy工作模式：

┌─────────────────────────────────────────────────────────────────────────┐
│                           kube-proxy                                     │
│                                                                         │
│  三种模式：                                                               │
│                                                                         │
│  1. userspace模式（已废弃）                                              │
│     ┌───────────────────────────────────────────────────────────────┐  │
│     │  Client → kube-proxy → iptables → Pod                        │  │
│     │              ↑                                                │  │
│     │         用户空间代理                                           │  │
│     │         性能较差                                               │  │
│     └───────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  2. iptables模式（默认）                                                 │
│     ┌───────────────────────────────────────────────────────────────┐  │
│     │  Client → iptables规则 → Pod                                  │  │
│     │              ↑                                                │  │
│     │         内核空间处理                                           │  │
│     │         性能好，规则多时更新慢                                  │  │
│     └───────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  3. IPVS模式（推荐）                                                     │
│     ┌───────────────────────────────────────────────────────────────┐  │
│     │  Client → IPVS规则 → Pod                                      │  │
│     │              ↑                                                │  │
│     │         内核空间负载均衡                                        │  │
│     │         性能最好，支持多种调度算法                               │  │
│     └───────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  Service类型支持：                                                        │
│  ├── ClusterIP：集群内部访问                                            │
│  ├── NodePort：通过节点端口访问                                         │
│  ├── LoadBalancer：云厂商负载均衡器                                     │
│  └── ExternalName：外部服务映射                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

```
│  │  │  │ Application │  │ Log Agent   │                      │    │   │
```

### Deployment

Deployment管理Pod的副本和更新策略。

```
Deployment架构：

┌─────────────────────────────────────────────────────────────────────────┐
│                          Deployment                                      │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                     Deployment                                   │   │
│  │  ├── replicas: 3                                                 │   │
│  │  ├── selector: app=myapp                                        │   │
│  │  └── strategy: RollingUpdate                                    │   │
│  └────────────────────────┬────────────────────────────────────────┘   │
│                           │                                             │
│                           ↓                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                     ReplicaSet                                   │   │
│  │  └── 管理Pod副本数量                                             │   │
│  └────────────────────────┬────────────────────────────────────────┘   │
│                           │                                             │
│                           ↓                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                               │
│  │  Pod 1  │  │  Pod 2  │  │  Pod 3  │                               │
│  └─────────┘  └─────────┘  └─────────┘                               │
│                                                                         │
│  更新策略：                                                               │
│  ├── RollingUpdate：滚动更新（默认）                                    │
│  │   ├── maxSurge：最多可超出副本数                                    │
│  │   └── maxUnavailable：最多不可用副本数                              │
│  └── Recreate：先删除后创建                                             │

│  kubectl rollout undo deployment/myapp                                 │
│  kubectl rollout history deployment/myapp                              │
```


## 常见问题

### Q1: Master节点可以运行工作负载吗？

可以，但不推荐：
- 通过去除master污点可以让Master运行Pod
- 生产环境建议Master专注于控制平面
- 小型测试集群可以混合部署

### Q2: etcd数据如何备份？

备份策略：
- 定期快照：etcdctl snapshot save
- 异地备份：备份文件存储到远程
- 验证恢复：定期演练恢复流程

### Q3: 如何选择容器运行时？

选择依据：
- containerd：生产环境推荐，稳定成熟
- CRI-O：轻量级，Kubernetes原生
- Docker：开发环境友好，已废弃支持

### Q4: Controller Manager和Scheduler如何高可用？

高可用方案：
- 多实例部署
- Leader选举机制
- 只有一个实例处于活跃状态

## 小结

Kubernetes核心组件协作实现容器编排：

**控制平面**：
- API Server：集群入口，认证授权
- etcd：数据存储，状态保存
- Scheduler：Pod调度
- Controller Manager：状态调谐

**工作节点**：
- Kubelet：节点代理，Pod管理
- kube-proxy：网络代理，负载均衡
- Container Runtime：容器运行

理解这些组件的职责和协作方式，是设计和管理Kubernetes集群的基础。

---

## 面试回答总结

Kubernetes的核心组件分为控制平面和工作节点两部分。控制平面包括：API Server作为集群的统一入口，处理认证、授权和准入控制，所有请求都通过它进入；etcd作为分布式键值存储，保存集群所有状态数据，建议3或5节点部署实现高可用；Scheduler负责Pod调度，通过预选和优选两个阶段选择最合适的节点；Controller Manager运行各种控制器，如Node Controller、Deployment Controller等，通过调谐循环确保实际状态与期望状态一致。工作节点包括：Kubelet作为节点代理，负责Pod的生命周期管理、健康检查和状态上报；kube-proxy维护网络规则，支持iptables和IPVS两种模式实现Service负载均衡；Container Runtime通过CRI接口运行容器，主流选择是containerd。这些组件通过watch机制监听API Server的变化，协同工作实现容器编排。
