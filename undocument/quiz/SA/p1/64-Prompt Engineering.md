# Prompt Engineering

## 问题:Prompt Engineering是什么?如何设计有效的提示词?

Prompt Engineering(提示工程)是设计和优化输入提示词以引导大模型生成期望输出的技术。它是与大模型交互的核心技能,通过精心设计的提示词,可以显著提升模型的输出质量和任务完成效果,是大模型应用的关键技术之一。

## Prompt设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                    Prompt设计六大原则                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  1. 清晰明确 (Clarity)                   │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  差的Prompt:                                             │   │
│  │  "写一篇文章"                                            │   │
│  │                                                         │   │
│  │  好的Prompt:                                             │   │
│  │  "请写一篇关于人工智能在教育领域应用的800字文章,        │   │
│  │   需要包含以下内容:                                      │   │
│  │   1. 人工智能在教育中的应用场景                          │   │
│  │   2. 面临的挑战和解决方案                                │   │
│  │   3. 未来发展趋势                                        │   │
│  │   语言风格: 专业但易懂,适合教育从业者阅读"              │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  2. 提供上下文 (Context)                 │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  差的Prompt:                                             │   │
│  │  "分析这个数据"                                          │   │
│  │                                                         │   │
│  │  好的Prompt:                                             │   │
│  │  "你是一位资深数据分析师,擅长电商数据分析。              │   │
│  │   请分析以下销售数据,找出主要趋势和异常点:              │   │
│  │   [数据内容]                                             │   │
│  │   分析维度:                                              │   │
│  │   - 销售趋势                                             │   │
│  │   - 用户行为                                             │   │
│  │   - 异常检测                                             │   │
│  │   请给出具体的洞察和建议"                                │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  3. 指定角色 (Role)                      │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  示例:                                                   │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ "你是一位资深Python开发工程师,              │       │   │
│  │  │  擅长编写高质量、可维护的代码。             │       │   │
│  │  │  请帮我优化以下代码,提高性能和可读性:"     │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  角色设定的作用:                                         │   │
│  │  · 激活模型相关领域的知识                               │   │
│  │  · 引导模型采用特定的语言风格                           │   │
│  │  · 提高输出的专业性                                     │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  4. 给出示例 (Examples)                  │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  Few-shot Prompting:                                     │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ 任务: 提取文本中的实体                      │       │   │
│  │  │                                             │       │   │
│  │  │ 示例1:                                      │       │   │
│  │  │ 输入: "苹果公司CEO库克访问了中国"           │       │   │
│  │  │ 输出: {"公司": "苹果公司", "人名": "库克", │       │   │
│  │  │        "地点": "中国"}                      │       │   │
│  │  │                                             │       │   │
│  │  │ 示例2:                                      │       │   │
│  │  │ 输入: "马斯克在德州建立了新工厂"            │       │   │
│  │  │ 输出: {"人名": "马斯克", "地点": "德州",   │       │   │
│  │  │        "事件": "建立新工厂"}                │       │   │
│  │  │                                             │       │   │
│  │  │ 现在请处理:                                 │       │   │
│  │  │ 输入: "张三在北京创办了一家科技公司"        │       │   │
│  │  │ 输出:                                       │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  5. 分步引导 (Step-by-step)              │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  Chain-of-Thought (思维链):                              │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ 问题: 小明有5个苹果,给了小红2个,           │       │   │
│  │  │       又买了3个,现在有几个苹果?             │       │   │
│  │  │                                             │       │   │
│  │  │ 请一步步思考:                               │       │   │
│  │  │ 步骤1: 小明最初有5个苹果                    │       │   │
│  │  │ 步骤2: 给了小红2个,剩下 5-2=3个            │       │   │
│  │  │ 步骤3: 又买了3个,现在有 3+3=6个            │       │   │
│  │  │ 答案: 6个苹果                               │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  作用: 引导模型展示推理过程,提高复杂任务的准确率         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  6. 指定输出格式 (Format)                │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  示例:                                                   │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ 请以JSON格式输出结果:                       │       │   │
│  │  │ {                                           │       │   │
│  │  │   "summary": "摘要内容",                    │       │   │
│  │  │   "keywords": ["关键词1", "关键词2"],       │       │   │
│  │  │   "sentiment": "positive/negative/neutral" │       │   │
│  │  │ }                                           │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  常见格式: JSON、Markdown表格、列表、XML等               │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 高级Prompt技术

```
┌─────────────────────────────────────────────────────────────────┐
│                    高级Prompt技术                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  1. Zero-shot vs Few-shot                │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  Zero-shot (零样本):                                     │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ "将以下句子翻译成英文:                      │       │   │
│  │  │  你好世界"                                  │       │   │
│  │  │                                             │       │   │
│  │  │ 优点: 简单直接                              │       │   │
│  │  │ 缺点: 复杂任务效果不佳                      │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  Few-shot (少样本):                                      │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ "将以下句子翻译成英文:                      │       │   │
│  │  │  示例1: 你好 → Hello                        │       │   │
│  │  │  示例2: 谢谢 → Thank you                    │       │   │
│  │  │  任务: 你好世界 → ?"                        │       │   │
│  │  │                                             │       │   │
│  │  │ 优点: 效果更好,任务理解更准确               │       │   │
│  │  │ 缺点: 需要准备示例,占用token                │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  2. Chain-of-Thought (CoT)               │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心思想: 让模型展示推理过程                            │   │
│  │                                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ 问题: 停车场有15辆车,开走了8辆,             │       │   │
│  │  │       又开进来12辆,现在有几辆车?             │       │   │
│  │  │                                             │       │   │
│  │  │ 让我们一步步思考:                            │       │   │
│  │  │ 1) 初始: 15辆车                              │       │   │
│  │  │ 2) 开走8辆后: 15 - 8 = 7辆                  │       │   │
│  │  │ 3) 开进12辆后: 7 + 12 = 19辆                │       │   │
│  │  │ 答案: 19辆车                                 │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  适用场景:                                               │   │
│  │  · 数学推理                                              │   │
│  │  · 逻辑推理                                              │   │
│  │  · 复杂决策                                              │   │
│  │  · 多步骤任务                                            │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  3. Tree-of-Thought (ToT)                │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心思想: 探索多个推理路径,选择最优解                   │   │
│  │                                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │           问题                              │       │   │
│  │  │             │                               │       │   │
│  │  │      ┌──────┼──────┐                       │       │   │
│  │  │      │      │      │                       │       │   │
│  │  │   思路1  思路2  思路3                       │       │   │
│  │  │      │      │      │                       │       │   │
│  │  │   评估1  评估2  评估3                       │       │   │
│  │  │      │      │      │                       │       │   │
│  │  │      └──────┼──────┘                       │       │   │
│  │  │             │                               │       │   │
│  │  │         最优解                              │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  适用场景: 创意写作、战略规划、复杂问题求解              │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  4. Self-Consistency                      │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心思想: 多次采样,选择最一致的答案                     │   │
│  │                                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │                                             │       │   │
│  │  │  问题 → 模型 → 答案1                        │       │   │
│  │  │       → 模型 → 答案2                        │       │   │
│  │  │       → 模型 → 答案3                        │       │   │
│  │  │       → 模型 → 答案4                        │       │   │
│  │  │       → 模型 → 答案5                        │       │   │
│  │  │                                             │       │   │
│  │  │  投票选择最常见答案                          │       │   │
│  │  │                                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  适用场景: 数学问题、逻辑推理、事实性问题                │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                  5. ReAct (Reasoning + Acting)           │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  核心思想: 推理与行动交替进行                            │   │
│  │                                                         │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │ 问题: 北京今天的天气如何?                   │       │   │
│  │  │                                             │       │   │
│  │  │ 思考1: 我需要查询北京今天的天气             │       │   │
│  │  │ 行动1: Search["北京天气 今天"]              │       │   │
│  │  │ 观察1: 北京今天晴,气温15-25度              │       │   │
│  │  │                                             │       │   │
│  │  │ 思考2: 我已经获得了天气信息                 │       │   │
│  │  │ 行动2: Finish["北京今天晴天,15-25度"]      │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                                                         │   │
│  │  适用场景: 需要外部工具、搜索、API调用的任务             │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│                    Prompt模板示例                                 │

│                                                                 │

│  │                                                         │   │
│  │  答案:                                                   │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │

│  │  {article}                                              │   │
│  │                                                         │   │
│  │  摘要要求:                                               │   │
│  │  - 长度: {length}字左右                                 │   │
│  │  - 风格: {style}                                        │   │
│  │  - 重点: {focus_points}                                 │   │
│  │                                                         │   │
│  │  摘要:                                                   │   │

## 常见问题

### Q1: Zero-shot和Few-shot如何选择?

选择Zero-shot还是Few-shot取决于任务复杂度和效果要求。Zero-shot适用于简单任务,如翻译、简单问答、基础分类等,优点是简单直接,不需要准备示例,缺点是复杂任务效果不佳。Few-shot适用于复杂任务,如特定格式的输出、专业领域的任务、需要遵循特定规则的任务等,优点是效果更好,模型更容易理解任务要求,缺点是需要准备高质量示例,占用更多token。建议先用Zero-shot测试,如果效果不理想再使用Few-shot,通常2-5个示例就能显著提升效果。

### Q2: 如何设计好的Few-shot示例?

设计Few-shot示例需要遵循几个原则:第一,代表性,示例要覆盖任务的主要情况,包括典型场景和边界情况;第二,多样性,示例之间要有差异,避免模型过度拟合某个模式;第三,质量,示例必须正确且高质量,错误的示例会误导模型;第四,格式一致,所有示例的输入输出格式要统一,便于模型学习;第五,数量适中,通常2-5个示例即可,太多会占用token且效果提升有限;第六,顺序合理,可以将相似示例放在一起,或按难度递进排列。实践中,建议从少量高质量示例开始,根据效果逐步调整。

### Q3: Chain-of-Thought什么时候使用?

Chain-of-Thought(CoT)适用于需要推理的任务,特别是:第一,数学问题,需要多步计算的问题;第二,逻辑推理,需要分析前提得出结论的问题;第三,复杂决策,需要权衡多个因素的问题;第四,问题分解,需要将复杂问题拆解为子问题的情况。CoT不适用于简单任务,如分类、翻译、简单问答等,反而会增加不必要的输出。使用CoT时,可以通过在Prompt中添加"让我们一步步思考"或"请详细说明推理过程"来引导模型展示推理步骤。对于特别复杂的问题,可以结合Few-shot CoT,提供带推理过程的示例。

### Q4: 如何评估Prompt的效果?

评估Prompt效果需要建立系统的评估流程:第一,准备测试集,包含多样化的测试用例,覆盖各种场景和边界情况;第二,定义评估指标,如准确率、相关性、流畅性、格式正确性等;第三,自动化评估,对于客观任务可以用脚本自动评分,如分类准确率、格式正确率;第四,人工评估,对于主观任务需要人工评分,可以制定评分标准保证一致性;第五,A/B测试,对比不同Prompt的效果,选择最优版本;第六,持续监控,上线后持续监控实际效果,收集bad case进行优化。建议建立Prompt版本管理,记录每次修改和效果变化。

### Q5: Prompt中的参数如何设置?

Prompt相关的参数主要包括:第一,temperature,控制输出随机性,0-1之间,值越低越确定,值越高越随机,事实性任务建议0-0.3,创意性任务建议0.7-1.0;第二,top_p,核采样参数,控制候选词范围,通常0.9-0.95,与temperature配合使用;第三,max_tokens,最大输出长度,根据任务需求设置,避免过长浪费或过短截断;第四,frequency_penalty,频率惩罚,减少重复内容,通常0-1;第五,presence_penalty,存在惩罚,鼓励多样性,通常0-1。建议从默认值开始,根据输出效果逐步调整。对于确定性任务,temperature设为0;对于创意任务,可以适当提高temperature和top_p。

## 小结

Prompt Engineering是设计和优化提示词以引导大模型生成期望输出的技术,包括六大设计原则:清晰明确、提供上下文、指定角色、给出示例、分步引导、指定输出格式。高级技术包括Zero-shot/Few-shot、Chain-of-Thought、Tree-of-Thought、Self-Consistency和ReAct等。设计Prompt时需要根据任务特点选择合适的技术,通过迭代优化提升效果。评估Prompt效果需要建立系统的评估流程,包括准备测试集、定义指标、自动化评估和人工评估。参数设置需要根据任务类型调整,确定性任务用低temperature,创意任务用高temperature。掌握Prompt Engineering是有效使用大模型的关键技能。

Prompt Engineering是设计和优化输入提示词以引导大模型生成期望输出的技术。核心设计原则包括六个方面:第一,清晰明确,明确说明任务要求和输出格式;第二,提供上下文,给模型足够的背景信息;第三,指定角色,如"你是一位资深工程师"来激活相关领域知识;第四,给出示例,Few-shot通过2-5个示例让模型更好理解任务;第五,分步引导,Chain-of-Thought让模型展示推理过程,提高复杂任务的准确率;第六,指定输出格式,如JSON、Markdown等。高级技术包括Zero-shot(无示例)和Few-shot(有示例)、CoT(思维链推理)、ToT(树状思维探索多路径)、Self-Consistency(多次采样投票)和ReAct(推理与行动交替)。评估效果需要准备测试集、定义指标、自动化和人工评估结合。参数设置上,确定性任务用低temperature(0-0.3),创意任务用高temperature(0.7-1.0)。Prompt Engineering是无需训练就能改变模型行为的低成本高效技术。
