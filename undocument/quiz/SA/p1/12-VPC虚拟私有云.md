# VPC虚拟私有云详解

## 什么是VPC？

VPC（Virtual Private Cloud，虚拟私有云）是在公有云中创建的逻辑隔离的虚拟网络环境。用户可以在VPC中启动云资源，并完全控制虚拟网络环境，包括IP地址范围、子网、路由表、网关等配置。

```
VPC概念示意：

┌─────────────────────────────────────────────────────────────────────────┐
│                          公有云环境                                      │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │                         其他用户VPC                                │  │
│  │  ┌─────────────────────────────────────────────────────────────┐  │  │
│  │  │  10.1.0.0/16                                                 │  │  │
│  │  └─────────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │                          您的VPC                                   │  │
│  │  ┌─────────────────────────────────────────────────────────────┐  │  │
│  │  │  CIDR: 10.0.0.0/16                                           │  │  │
│  │  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │  │  │
│  │  │  │ 公有子网    │  │ 私有子网    │  │ 数据库子网  │         │  │  │
│  │  │  │ 10.0.1.0/24 │  │ 10.0.2.0/24 │  │ 10.0.3.0/24 │         │  │  │
│  │  │  └─────────────┘  └─────────────┘  └─────────────┘         │  │  │
│  │  │                                                              │  │  │
│  │  │  逻辑隔离，完全控制网络环境                                   │  │  │
│  │  └─────────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────────┘  │
```

## VPC核心组件

### 1. CIDR和IP地址规划

```

┌─────────────────────────────────────────────────────────────────────────┐

│  │                                                                 │   │
│  │  子网划分：                                                      │   │
│  │  ├── 公有子网A (AZ-1): 10.0.1.0/24    256个IP                   │   │

│  │  ├── 数据库子网A:     10.0.21.0/24    256个IP                   │   │
```

### 4. 安全组和网络ACL

```
安全组 vs 网络ACL：

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  安全组（Security Group）：                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  特点：                                                         │   │
│  │  ├── 实例级别（虚拟防火墙）                                     │   │
│  │  ├── 仅允许规则（白名单）                                       │   │
│  │  ├── 有状态（自动允许响应流量）                                 │   │
│  │  └── 可引用其他安全组                                           │   │
│  │                                                                 │   │
│  │  示例配置：                                                     │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │  Web服务器安全组                                         │   │   │
│  │  │  入站规则：                                               │   │   │
│  │  │  ├── 80/HTTP   来自 0.0.0.0/0                           │   │   │
│  │  │  ├── 443/HTTPS 来自 0.0.0.0/0                           │   │   │
│  │  │  └── 22/SSH    来自 Bastion安全组                       │   │   │
│  │  │  出站规则：                                               │   │   │
│  │  │  └── 全部      到 0.0.0.0/0                              │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  网络ACL（Network ACL）：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  特点：                                                         │   │
│  │  ├── 子网级别                                                   │   │
│  │  ├── 允许和拒绝规则                                             │   │
│  │  ├── 无状态（需要配置双向规则）                                 │   │
│  │  └── 按规则编号顺序处理                                         │   │
│  │                                                                 │   │
│  │  示例配置：                                                     │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │  私有子网网络ACL                                          │   │   │
│  │  │  入站规则：                                               │   │   │
│  │  │  ├── 100  允许 10.0.1.0/24 80/HTTP                      │   │   │
│  │  │  ├── 110  允许 10.0.1.0/24 443/HTTPS                    │   │   │

│  │  │  出站规则：                                               │   │   │
│  │  │  ├── 100  允许 10.0.1.0/24 1024-65535                   │   │   │
│  │  │  └── *    拒绝 0.0.0.0/0     所有                       │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  对比：                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  特性              安全组              网络ACL                   │   │
│  │  ─────────────────────────────────────────────────────────────  │   │
│  │  层级              实例级              子网级                    │   │
│  │  规则类型          仅允许              允许和拒绝                │   │
│  │  状态              有状态              无状态                    │   │
│  │  规则顺序          无影响              按编号顺序                │   │
│  │  推荐用途          主要防护            补充防护                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3. VPC对等连接

```
VPC对等连接：

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  ┌─────────────────┐                    ┌─────────────────┐            │
│  │    VPC-A        │                    │    VPC-B        │            │
│  │  10.0.0.0/16    │ ←── 对等连接 ───→  │  10.1.0.0/16    │            │
│  │  ┌───────────┐  │                    │  ┌───────────┐  │            │
│  │  │ 应用服务  │  │                    │  │ 共享服务  │  │            │
│  │  └───────────┘  │                    │  └───────────┘  │            │
│  └─────────────────┘                    └─────────────────┘            │
│                                                                         │
│  特点：                                                                   │
│  ├── 两个VPC之间直接连接                                                │
│  ├── 不支持传递性（A-B-C，A不能通过B访问C）                             │
│  ├── CIDR不能重叠                                                       │
```

## 常见问题

### Q1: VPC和传统网络有什么区别？

区别：
- **VPC**：逻辑隔离，软件定义，弹性扩展
- **传统网络**：物理隔离，硬件设备，固定容量

### Q2: 如何选择公有子网还是私有子网？

选择依据：
- **公有子网**：需要公网访问的服务（ALB、NAT、堡垒机）
- **私有子网**：内部服务（应用服务器、数据库）

### Q3: 安全组和网络ACL如何配合使用？

最佳实践：
- 安全组作为主要防护层
- 网络ACL作为补充防护层
- 安全组允许必要流量
- 网络ACL拒绝已知威胁

### Q4: VPC如何实现高可用？

高可用策略：
- 多可用区部署
- 跨可用区子网
- 冗余网关
- 路由表冗余

## 小结

VPC是云网络的基础，核心组件包括：
- **CIDR**：IP地址规划
- **子网**：网络分段
- **路由表**：流量路由
- **安全组/网络ACL**：访问控制

VPC连接选项：
- **Internet网关**：公网访问
- **NAT网关**：私有子网访问公网
- **VPC对等**：VPC互联
- **VPN/专线**：混合云连接

## 面试回答总结

VPC（虚拟私有云）是在公有云中创建的逻辑隔离虚拟网络环境，用户可以完全控制IP地址范围、子网划分、路由表配置和网关设置。VPC的核心组件包括：CIDR用于IP地址规划，建议使用RFC1918私有地址空间并预留扩展空间；子网分为公有子网（有Internet网关路由，适合面向公网的服务）和私有子网（无直接公网路由，适合内部服务）；路由表控制流量路由方向；安全组是实例级有状态防火墙，网络ACL是子网级无状态防火墙。VPC的网络隔离作用体现在：通过子网划分实现不同安全级别的网络分段，通过安全组和网络ACL实现多层访问控制，通过路由表控制流量走向。VPC连接选项包括：Internet网关用于公网访问，NAT网关让私有子网可以访问互联网，VPC对等连接实现VPC间互联，VPN和专线实现混合云连接。设计VPC时需要考虑多可用区部署实现高可用，合理规划IP地址避免冲突，以及分层安全策略。
