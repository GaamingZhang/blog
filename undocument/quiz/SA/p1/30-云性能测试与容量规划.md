# 云性能测试与容量规划

## 为什么需要性能测试与容量规划？

想象你经营着一家餐厅。如果座位太少，客人来了没位置，损失收入；如果座位太多，平时空着浪费成本。你需要根据客流量预测来合理安排座位数量和服务人员。

云性能测试与容量规划就像餐厅的"客流预测"——通过测试了解系统性能极限，根据业务预测规划资源容量，确保既能应对高峰流量，又不会过度浪费成本。

│  │     ├── 场景：测试数据库、存储等大数据量场景                    │   │
│  │     ├── 指标：数据处理能力、存储性能                            │   │
│  │     └── 示例：测试百万级数据查询性能                            │   │

### k6测试脚本示例

```javascript
// k6性能测试脚本示例
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Rate } from 'k6/metrics';

// 自定义指标
const errorRate = new Rate('errors');

// 测试配置
export const options = {
  stages: [
    // 阶段1：预热，逐步增加到100用户
    { duration: '2m', target: 100 },
    // 阶段2：正常负载，保持100用户
    { duration: '5m', target: 100 },
    // 阶段3：峰值负载，增加到500用户
    { duration: '2m', target: 500 },
    // 阶段4：峰值保持
    { duration: '5m', target: 500 },
    // 阶段5：逐步减少
    { duration: '2m', target: 0 },
  ],
  thresholds: {
    // 响应时间阈值
    http_req_duration: ['p(95)<200', 'p(99)<500'],
    // 错误率阈值
    errors: ['rate<0.01'],
    // HTTP失败率阈值
    http_req_failed: ['rate<0.01'],
  },
};

// 测试场景
export default function () {
  // 登录请求
  const loginRes = http.post('https://api.example.com/login', JSON.stringify({
    username: 'testuser',
    password: 'testpass',
  }), {
    headers: { 'Content-Type': 'application/json' },
  });

  // 检查登录结果
  check(loginRes, {
    'login successful': (r) => r.status === 200,
    'has token': (r) => r.json('token') !== '',
  });

  // 记录错误
  errorRate.add(loginRes.status !== 200);

  // 获取用户列表
  const usersRes = http.get('https://api.example.com/users', {
    headers: { 
      'Authorization': `Bearer ${loginRes.json('token')}` 
    },
  });

  check(usersRes, {
    'users status is 200': (r) => r.status === 200,
    'users response time < 200ms': (r) => r.timings.duration < 200,
  });

  errorRate.add(usersRes.status !== 200);

  // 思考时间
  sleep(1);
}
```

## 常见问题

### Q1: 如何确定性能测试的场景？

**场景设计**：
- 分析业务流程
- 识别关键路径
- 确定并发用户数
- 设置思考时间
- 设计数据量

### Q2: 性能测试应该在什么环境进行？

**环境选择**：
- 开发环境：快速验证
- 测试环境：功能测试
- 预生产环境：接近生产
- 生产环境：真实流量测试（谨慎）

### Q3: 如何处理性能瓶颈？

**处理流程**：
1. 监控识别瓶颈
2. 分析瓶颈原因
3. 制定优化方案
4. 实施优化
5. 验证效果

### Q4: 如何进行容量预测？

**预测方法**：
- 历史数据分析
- 业务增长预测
- 季节性因素
- 营销活动影响
- 行业趋势

### Q5: 如何平衡性能和成本？

**平衡策略**：
- 合理设置SLA
- 使用弹性伸缩
- 优化资源利用率
- 选择合适的实例类型
- 使用预留实例和折扣

## 小结

云性能测试与容量规划是确保系统稳定性和成本效益的关键实践。

**性能测试**：
1. 类型：负载测试、压力测试、峰值测试、浸泡测试
2. 指标：响应时间、吞吐量、并发数、错误率、资源使用率
3. 工具：JMeter、Gatling、Locust、k6

**容量规划**：
1. 流程：需求分析→基准测试→容量计算→架构设计→成本估算
2. 计算：基于性能基准和业务需求
3. 优化：应用层、基础设施层

**关键原则**：
- 持续测试和监控
- 预留容量缓冲
- 弹性伸缩设计
- 成本效益平衡

## 面试回答总结

云性能测试与容量规划包括性能测试和容量规划两个方面。性能测试类型包括负载测试（验证系统在预期负载下的性能）、压力测试（找到系统性能极限和崩溃点）、峰值测试（验证系统应对突发流量的能力）、浸泡测试（验证系统长期运行的稳定性）、容量测试（验证系统处理大量数据的能力）。关键性能指标包括响应时间（P50、P95、P99）、吞吐量（QPS、TPS）、并发数、错误率、资源使用率（CPU、内存、磁盘I/O、网络带宽）。主流性能测试工具包括JMeter（功能强大、插件丰富）、Gatling（高性能、代码化测试）、Locust（Python编写、易扩展）、k6（Go编写、高性能、云原生友好）。容量规划流程包括业务需求分析（用户规模预测、业务增长预测、峰值流量预估、SLA要求）、性能基准测试（单实例性能基准、资源使用基准、瓶颈识别、性能模型建立）、容量计算（实例数量计算、资源规格选择、存储容量规划、网络带宽规划）、架构设计（高可用设计、弹性伸缩设计、缓存策略、数据库分片）、成本估算（资源成本、流量成本、存储成本、总体成本）、持续优化（监控告警、定期压测、容量调整、成本优化）。容量计算示例：目标QPS 10000、单实例QPS 500、冗余系数1.3，则最小实例数=10000/500×1.3=26个。性能优化策略包括应用层优化（代码优化、缓存策略、数据库优化、架构优化）和基础设施优化（计算资源优化、存储优化、网络优化、容器化优化）。
