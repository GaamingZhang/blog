# API网关设计

## 什么是API网关？

想象你经营着一家大型购物中心。传统方式下，顾客需要分别进入每个商店，每个商店都有自己的入口、保安、收银台。这导致管理混乱、体验糟糕。

API网关就像购物中心的"统一入口大厅"——顾客只需要从一个入口进入，由统一的前台提供导航、安检、结算等服务，然后再引导到各个商店。这样既提升了顾客体验，又便于统一管理。

在微服务架构中，API网关是系统的统一入口，负责请求路由、协议转换、认证授权、限流熔断等功能，将客户端与后端服务解耦。

## 常见问题

### Q1: API网关会成为性能瓶颈吗？

**可能的瓶颈**：
- 单点故障风险
- 性能压力集中
- 网络延迟增加

**解决方案**：
- 高可用部署（多节点、负载均衡）
- 水平扩展
- 缓存策略
- 异步处理

### Q2: 如何处理跨域问题？

**解决方案**：
```yaml
# CORS配置
plugins:
  - name: cors
    config:
      origins:
        - "https://example.com"
      methods:
        - GET
        - POST
      headers:
        - Accept
        - Authorization
        - Content-Type
      exposed_headers:
        - X-Auth-Token
      credentials: true
      max_age: 3600
```

### Q3: API网关如何实现灰度发布？

**实现方式**：
1. 基于请求头的路由
2. 基于权重的流量分配
3. 基于用户ID的分流

```yaml
# 灰度发布配置示例
routes:
  - name: canary-route
    paths:
      - /api/v1/users
    plugins:
      - name: canary
        config:
          percentage: 10  # 10%流量到新版本
          upstream_host: user-service-v2
```

### Q4: API网关与服务网格如何配合？

**配合方式**：
- API网关：处理外部流量入口
- Service Mesh：处理内部服务间通信
- 网关作为Mesh的一部分或独立部署

### Q5: 如何监控API网关？

**监控指标**：
- 请求数量、错误率、响应时间
- 上游服务健康状态
- 限流、熔断触发次数
- 资源使用情况（CPU、内存）

**工具**：
- Prometheus + Grafana
- ELK Stack
- Jaeger/Zipkin

## 小结

API网关是微服务架构的关键基础设施，承担着统一入口、流量管理、安全防护、协议转换等核心职责。

**核心功能**：
1. 路由与负载均衡：请求分发、服务发现
2. 认证与授权：统一身份验证、权限控制
3. 限流与熔断：保护系统免受流量冲击
4. 协议转换：HTTP/gRPC转换、请求聚合
5. 可观测性：日志、监控、追踪

**设计要点**：
1. 高可用：多节点部署、负载均衡
2. 高性能：缓存、异步处理
3. 可扩展：插件机制、动态配置
4. 安全性：认证授权、加密传输

**选型建议**：
- Spring Cloud生态：Spring Cloud Gateway
- 高性能场景：Kong、Nginx
- 云原生环境：APISIX、Envoy
- AWS云上：AWS API Gateway

## 面试回答总结

API网关是微服务架构的统一入口，负责请求路由、认证授权、限流熔断、协议转换等功能。设计API网关需要考虑以下关键因素：首先是核心功能，包括路由与负载均衡（路径匹配、服务发现、负载均衡策略）、认证与授权（API Key、JWT、OAuth 2.0等认证方式，RBAC/ABAC授权模型）、限流与熔断（令牌桶、漏桶等限流算法，熔断器状态机）、协议转换（HTTP/gRPC转换、请求聚合）、可观测性（日志、监控、追踪）。其次是架构设计，需要保证高可用（多节点部署、负载均衡、无状态设计）、高性能（缓存、异步处理）、可扩展（插件机制、动态配置）。主流API网关包括Kong（基于Nginx+Lua，插件丰富）、Spring Cloud Gateway（Java生态友好）、APISIX（云原生，动态配置）、AWS API Gateway（托管服务）。选型时需要考虑技术栈、性能需求、功能需求、运维成本等因素。API网关可能成为性能瓶颈，需要通过水平扩展、缓存策略、异步处理等方式解决。
