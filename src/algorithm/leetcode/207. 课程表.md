# 207. 课程表

你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1 。
在选修某些课程之前需要一些先修课程。 先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，
表示如果要学习课程 ai 则 必须 先学习课程  bi 。
例如，先修课程对 [0, 1] 表示：想要学习课程 0 ，你需要先完成课程 1 。
请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。

示例 1：
输入：numCourses = 2, prerequisites = [[1,0]]
输出：true
解释：总共有 2 门课程。学习课程 1 之前，你需要完成课程 0 。这是可能的。

示例 2：
输入：numCourses = 2, prerequisites = [[1,0],[0,1]]
输出：false
解释：总共有 2 门课程。学习课程 1 之前，你需要先完成​课程 0 ；并且学习课程 0 之前，
 你还应先完成课程 1 。这是不可能的。

## 解题思路

这道题可以转化为**判断有向图中是否存在环**的问题。如果存在环，则无法完成所有课程。

### 拓扑排序

使用拓扑排序判断是否有环：

1. 构建有向图和入度数组
2. 将所有入度为 0 的节点加入队列
3. 依次处理队列中的节点，减少其邻居的入度
4. 如果邻居入度变为 0，加入队列
5. 统计处理的节点数量，如果等于课程数则无环

### 算法流程

1. **构建图**：根据先修关系构建邻接表
2. **计算入度**：统计每个节点的入度
3. **拓扑排序**：
   - 入度为 0 的节点入队
   - 处理节点，减少邻居入度
   - 入度变为 0 的邻居入队
4. **判断结果**：处理的节点数 == 课程数则无环

### 示例分析

`numCourses = 2, prerequisites = [[1,0]]`：
- 图：0 → 1
- 入度：[0, 1]
- 处理 0，入度变为 [0, 0]，1 入队
- 处理 1，count = 2
- 结果：true

`numCourses = 2, prerequisites = [[1,0],[0,1]]`：
- 图：0 → 1, 1 → 0（环）
- 入度：[1, 1]
- 无入度为 0 的节点
- count = 0
- 结果：false

## 代码实现

```cpp
class Solution {
public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        // 邻接表：存储图
        vector<vector<int>> graph(numCourses);
        // 入度数组
        vector<int> indegree(numCourses, 0);
        
        // 构建图和计算入度
        for (const auto& prereq : prerequisites) {
            int a = prereq[0];  // 后修课程
            int b = prereq[1];  // 先修课程
            graph[b].emplace_back(a);  // b -> a
            indegree[a]++;  // a 的入度加 1
        }
        
        // 将入度为 0 的节点加入队列
        queue<int> que;
        for (int i = 0; i < numCourses; i++) {
            if (indegree[i] == 0) {
                que.push(i);
            }
        }
        
        int count = 0;  // 统计已处理的节点数
        
        // 拓扑排序
        while (!que.empty()) {
            int node = que.front();
            que.pop();
            count++;  // 处理一个节点
            
            // 减少邻居的入度
            for (int neighbor : graph[node]) {
                indegree[neighbor]--;
                // 如果入度变为 0，加入队列
                if (indegree[neighbor] == 0) {
                    que.push(neighbor);
                }
            }
        }
        
        // 如果处理的节点数等于课程数，说明无环
        return count >= numCourses;
    }
};
```

## 复杂度分析

- **时间复杂度**：O(V + E)，其中 V 是课程数，E 是先修关系数。需要遍历所有节点和边。
- **空间复杂度**：O(V + E)，邻接表和入度数组的空间。

## 总结

本题使用**拓扑排序**判断课程安排是否可行。关键在于：

1. **图建模**：将课程关系转化为有向图
2. **入度计算**：统计每个节点的入度
3. **拓扑排序**：通过入度判断是否有环

其他解法：
- **DFS**：使用深度优先搜索检测环
