# 135. 分发糖果

n 个孩子站成一排。给你一个整数数组 ratings 表示每个孩子的评分。
你需要按照以下要求，给这些孩子分发糖果：
每个孩子至少分配到 1 个糖果。
相邻两个孩子评分更高的孩子会获得更多的糖果。
请你给每个孩子分发糖果，计算并返回需要准备的 最少糖果数目 。

示例 1：
输入：ratings = [1,0,2]
输出：5
解释：你可以分别给第一个、第二个、第三个孩子分发 2、1、2 颗糖果。

示例 2：
输入：ratings = [1,2,2]
输出：4
解释：你可以分别给第一个、第二个、第三个孩子分发 1、2、1 颗糖果。
第三个孩子只得到 1 颗糖果，因为满足题目要求，但评分与第二个孩子相同，不需要更多糖果。

## 解题思路

这道题需要满足两个方向的约束条件：
1. 如果右边评分更高，右边糖果要比左边多
2. 如果左边评分更高，左边糖果要比右边多

### 两次遍历法

由于需要同时满足左右两个方向的约束，可以分两次遍历：

1. **从左到右遍历**：满足右边评分高则右边糖果多的约束
2. **从右到左遍历**：满足左边评分高则左边糖果多的约束
3. **取最大值**：每个位置的糖果数取两次遍历结果的最大值

### 为什么需要两次遍历？

如果只从左到右遍历，只能保证右边评分高时右边糖果多，但无法保证左边评分高时左边糖果多。例如 `[1, 2, 87, 87, 87, 2, 1]`，从左到右遍历后，中间的 87 会得到很多糖果，但右边的 2 和 1 无法正确处理。

### 算法流程

1. 初始化所有孩子的糖果数为 1
2. 从左到右遍历：如果右边评分更高，右边糖果 = 左边糖果 + 1
3. 从右到左遍历：如果左边评分更高，左边糖果 = max(左边糖果, 右边糖果 + 1)
4. 累加所有糖果数

## 代码实现

```cpp
class Solution {
public:
    int candy(vector<int>& ratings) {
        int n = ratings.size();
        // 初始化每个孩子至少 1 颗糖果
        vector<int> nums(n, 1);
        
        // 从左到右遍历：处理右边评分更高的情况
        for (int i = 0; i < n - 1; ++i) {
            if (ratings[i] < ratings[i + 1]) {
                // 右边评分更高，右边糖果 = 左边糖果 + 1
                nums[i + 1] = nums[i] + 1;
            }
            // 评分相等或左边更高，不处理（保持至少 1 颗）
        }
        
        // 从右到左遍历：处理左边评分更高的情况
        for (int i = n - 1; i > 0; --i) {
            if (ratings[i] < ratings[i - 1]) {
                // 左边评分更高，左边糖果 = max(当前值, 右边糖果 + 1)
                nums[i - 1] = max(nums[i - 1], nums[i] + 1);
            }
            // 评分相等或右边更高，不处理
        }
        
        // 计算总糖果数
        int sum = 0;
        for (int i = 0; i < n; ++i) {
            sum += nums[i];
        }
        
        return sum;
    }
};
```

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 是孩子数量。需要遍历两次数组。
- **空间复杂度**：O(n)，需要额外的数组存储每个孩子的糖果数。

## 总结

本题使用**两次遍历法**解决双向约束问题。关键在于：

1. **分解约束**：将双向约束分解为两个单向约束
2. **从左到右**：处理右边评分更高的情况
3. **从右到左**：处理左边评分更高的情况
4. **取最大值**：保证同时满足两个方向的约束

这种方法是贪心算法的经典应用，通过局部最优达到全局最优。
