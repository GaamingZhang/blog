# 219. 存在重复元素 II

给你一个整数数组 nums 和一个整数 k ，判断数组中是否存在两个 不同的索引 i 和 j ，满足
 nums[i] == nums[j] 且 abs(i - j) <= k 。如果存在，返回 true ；否则，返回 false 。

示例 1：
输入：nums = [1,2,3,1], k = 3
输出：true

示例 2：
输入：nums = [1,0,1,1], k = 1
输出：true

示例 3：
输入：nums = [1,2,3,1,2,3], k = 2
输出：false

## 解题思路

这道题要求判断数组中是否存在两个相同的元素，且它们的索引差不超过 k。

### 方法一：哈希表

使用哈希表记录每个元素最近出现的索引位置。遍历数组时：
1. 如果当前元素已在哈希表中，检查索引差是否 <= k
2. 更新当前元素的索引为最新位置

### 方法二：滑动窗口 + 哈希集合

维护一个大小为 k 的滑动窗口：
1. 窗口内使用哈希集合存储元素
2. 遍历时，如果当前元素在窗口内存在，返回 true
3. 将当前元素加入窗口，如果窗口大小超过 k，移除最左边的元素

## 代码实现

### 方法一：哈希表

```cpp
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        unordered_map<int, int> lastIndex;
        
        for (int i = 0; i < nums.size(); ++i) {
            if (lastIndex.find(nums[i]) != lastIndex.end()) {
                if (i - lastIndex[nums[i]] <= k) {
                    return true;
                }
            }
            lastIndex[nums[i]] = i;
        }
        
        return false;
    }
};
```

### 方法二：滑动窗口

```cpp
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        unordered_set<int> window;
        
        for (int i = 0; i < nums.size(); ++i) {
            if (window.find(nums[i]) != window.end()) {
                return true;
            }
            
            window.insert(nums[i]);
            
            if (window.size() > k) {
                window.erase(nums[i - k]);
            }
        }
        
        return false;
    }
};
```

### 代码详解

**方法一：哈希表**

1. `lastIndex`：记录每个元素最近出现的索引
2. 遍历数组，对于每个元素：
   - 如果元素已存在，检查 `i - lastIndex[nums[i]] <= k`
   - 更新元素的最新索引
3. 时间复杂度 O(n)，空间复杂度 O(n)

**方法二：滑动窗口**

1. `window`：维护一个大小不超过 k 的窗口
2. 遍历数组，对于每个元素：
   - 如果元素在窗口内存在，返回 true
   - 将元素加入窗口
   - 如果窗口大小超过 k，移除 `nums[i-k]`
3. 时间复杂度 O(n)，空间复杂度 O(k)

**示例演示**（nums = [1,2,3,1], k = 3）：
```
方法一：
i=0: nums[0]=1, lastIndex={}, 更新 lastIndex[1]=0
i=1: nums[1]=2, lastIndex={1:0}, 更新 lastIndex[2]=1
i=2: nums[2]=3, lastIndex={1:0,2:1}, 更新 lastIndex[3]=2
i=3: nums[3]=1, lastIndex[1]=0, 3-0=3 <= 3, 返回 true

方法二：
i=0: window={}, 插入1, window={1}
i=1: window={1}, 插入2, window={1,2}
i=2: window={1,2}, 插入3, window={1,2,3}
i=3: window={1,2,3}, 1在window中, 返回 true
```

## 复杂度分析

**方法一：哈希表**
- 时间复杂度：O(n)，遍历一次数组
- 空间复杂度：O(n)，最坏情况存储所有元素

**方法二：滑动窗口**
- 时间复杂度：O(n)，遍历一次数组
- 空间复杂度：O(k)，窗口最多存储 k+1 个元素

## 总结

本题考察的是哈希表和滑动窗口的应用：

1. **哈希表方法**：记录每个元素的最近位置，适合需要知道具体索引差的场景
2. **滑动窗口方法**：维护固定大小的窗口，空间复杂度更优

两种方法都能高效解决问题，滑动窗口方法在空间上更优，特别是当 k 远小于 n 时。选择哪种方法取决于具体场景和约束条件。
