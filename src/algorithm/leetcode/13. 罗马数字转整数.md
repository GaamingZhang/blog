# 13. 罗马数字转整数

罗马数字包含以下七种字符: I， V， X， L，C，D 和 M。
字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000
例如， 罗马数字 2 写做 II ，即为两个并列的 1 。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII， 而是 IV。数字 1 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地， 数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：

I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。
C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。
给定一个罗马数字，将其转换成整数。

示例 1:
输入: s = "III"
输出：3

示例 2:
输入：s = "IV"
输出：4

## 解题思路

这道题是上一题的逆过程，需要将罗马数字转换为整数。

### 关键规则

罗马数字的转换规则可以简化为：
- **加法**：如果当前字符的值 >= 下一个字符的值，则加上当前值
- **减法**：如果当前字符的值 < 下一个字符的值，则减去当前值

### 从右向左遍历

从右向左遍历字符串，可以简化判断逻辑：
1. 最右边的字符总是加到结果中
2. 如果当前字符的值 >= 上一个字符的值，则加上当前值
3. 如果当前字符的值 < 上一个字符的值，则减去当前值

**示例分析**：`MCMXCIV`（1994）
- V = 5，result = 5
- I = 1 < 5，result = 5 - 1 = 4
- C = 100 > 1，result = 4 + 100 = 104
- X = 10 < 100，result = 104 - 10 = 94
- M = 1000 > 10，result = 94 + 1000 = 1094
- C = 100 < 1000，result = 1094 - 100 = 994
- M = 1000 > 100，result = 994 + 1000 = 1994

## 代码实现

```cpp
class Solution {
public:
    int romanToInt(string s) {
        // 从右向左遍历，最右边的字符总是加到结果中
        int result = nums[s[s.size() - 1]];
        
        // 从倒数第二个字符开始向左遍历
        for (int i = s.size() - 2; i >= 0; --i) {
            // 如果当前字符值 >= 下一个字符值，则加上当前值
            // 否则减去当前值（减法形式，如 IV、IX）
            if (nums[s[i]] >= nums[s[i + 1]]) {
                result += nums[s[i]];
            } else {
                result -= nums[s[i]];
            }
        }
        
        return result;
    }

private:
    // 罗马数字字符到数值的映射
    unordered_map<char, int> nums = {
        {'I', 1},
        {'V', 5},
        {'X', 10},
        {'L', 50},
        {'C', 100},
        {'D', 500},
        {'M', 1000}
    };
};
```

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 是字符串长度。只需遍历一次字符串。
- **空间复杂度**：O(1)，哈希表大小固定为 7。

## 总结

本题的核心是理解罗马数字的**加法规则和减法规则**。从右向左遍历可以简化判断逻辑：当前字符如果比右边字符小，则表示减法形式。

这种方法的优点是只需一次遍历，逻辑简洁清晰。
