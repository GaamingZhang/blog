# 918. 环形子数组的最大和

给定一个长度为 n 的环形整数数组 nums ，返回 nums 的非空 子数组 的最大可能和 。

环形数组 意味着数组的末端将会与开头相连呈环状。形式上， nums[i] 的下一个元素是 nums[(i + 1) % n] ， nums[i] 的前一个元素是 nums[(i - 1 + n) % n] 。

子数组 最多只能包含固定缓冲区 nums 中的每个元素一次。形式上，对于子数组 nums[i], nums[i + 1], ..., nums[j] ，不存在 i <= k1, k2 <= j 其中 k1 % n == k2 % n 。

示例 1：
输入：nums = [1,-2,3,-2]
输出：3
解释：从子数组 [3] 得到最大和 3

示例 2：
输入：nums = [5,-3,5]
输出：10
解释：从子数组 [5,5] 得到最大和 5 + 5 = 10

示例 3：
输入：nums = [3,-2,2,-3]
输出：3
解释：从子数组 [3] 和 [3,-2,2] 都可以得到最大和 3

## 解题思路

### 问题分析

环形数组的最大子数组和有两种情况：

1. **不跨越边界**：最大子数组在数组中间，与普通数组相同
2. **跨越边界**：最大子数组包含首尾元素，跨越数组边界

### 核心思想

对于跨越边界的情况，可以转化为：
- 最大子数组和 = 数组总和 - 最小子数组和

因此，最终结果为：
- `max(不跨越的最大子数组和, 数组总和 - 最小子数组和)`

### 特殊情况

如果所有元素都是负数，那么 `sum - gmin = 0`，此时应该返回 `fmax`（最大的那个负数）。

### Kadane 算法

使用 Kadane 算法同时计算：
- `fmax`：最大子数组和
- `gmin`：最小子数组和
- `sum`：数组总和

## 代码实现

```cpp
class Solution {
public:
    int maxSubarraySumCircular(vector<int>& nums) {
        int f = nums[0];      // 当前最大子数组和
        int fmax = nums[0];   // 全局最大子数组和
        int g = nums[0];      // 当前最小子数组和
        int gmin = nums[0];   // 全局最小子数组和
        int sum = nums[0];    // 数组总和
        
        for (int i = 1; i < nums.size(); ++i) {
            f = max(f + nums[i], nums[i]);
            fmax = max(fmax, f);
            g = min(g + nums[i], nums[i]);
            gmin = min(gmin, g);
            sum += nums[i];
        }
        
        return sum == gmin ? fmax : max(fmax, sum - gmin);
    }
};
```

### 代码详解

**变量说明**：
- `f`：以当前位置结尾的最大子数组和
- `fmax`：全局最大子数组和（不跨越边界的情况）
- `g`：以当前位置结尾的最小子数组和
- `gmin`：全局最小子数组和
- `sum`：数组总和

**状态转移**：
```cpp
f = max(f + nums[i], nums[i]);  // 最大子数组和
fmax = max(fmax, f);
g = min(g + nums[i], nums[i]);  // 最小子数组和
gmin = min(gmin, g);
```

**结果判断**：
```cpp
return sum == gmin ? fmax : max(fmax, sum - gmin);
```
- 如果 `sum == gmin`，说明所有元素都是负数，返回 `fmax`
- 否则返回 `max(fmax, sum - gmin)`

**示例演示**（nums = [5,-3,5]）：
```
初始：f=5, fmax=5, g=5, gmin=5, sum=5

i=1, nums[1]=-3:
  f = max(5-3, -3) = 2
  fmax = max(5, 2) = 5
  g = min(5-3, -3) = -3
  gmin = min(5, -3) = -3
  sum = 5 + (-3) = 2

i=2, nums[2]=5:
  f = max(2+5, 5) = 7
  fmax = max(5, 7) = 7
  g = min(-3+5, 5) = 2
  gmin = min(-3, 2) = -3
  sum = 2 + 5 = 7

sum(7) != gmin(-3)
结果：max(7, 7-(-3)) = max(7, 10) = 10

跨越边界的情况：[5,5] = 5 + 5 = 10
```

## 复杂度分析

**时间复杂度**：O(n)，遍历一次数组

**空间复杂度**：O(1)，只使用常数额外空间

## 总结

本题将环形数组问题转化为普通数组问题：

1. **两种情况**：不跨越边界和跨越边界
2. **转化思路**：跨越边界的最大和 = 总和 - 最小子数组和
3. **特殊情况**：全负数时返回最大元素

这种"正难则反"的思想在解决环形问题时很常见，类似的还有环形房屋打劫等问题。理解如何将环形问题转化为线性问题是解决此类问题的关键。
