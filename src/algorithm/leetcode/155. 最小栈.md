# 155. 最小栈

设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈。
实现 MinStack 类:
MinStack() 初始化堆栈对象。
void push(int val) 将元素val推入堆栈。
void pop() 删除堆栈顶部的元素。
int top() 获取堆栈顶部的元素。
int getMin() 获取堆栈中的最小元素。

示例 1:
输入：
["MinStack","push","push","push","getMin","pop","top","getMin"]
[[],[-2],[0],[-3],[],[],[],[]]

输出：
[null,null,null,null,-3,null,0,-2]

解释：
MinStack minStack = new MinStack();
minStack.push(-2);
minStack.push(0);
minStack.push(-3);
minStack.getMin();   --> 返回 -3.
minStack.pop();
minStack.top();      --> 返回 0.
minStack.getMin();   --> 返回 -2.

## 解题思路

这道题需要设计一个能在 O(1) 时间内获取最小值的栈。

### 辅助栈法

使用两个栈：
1. **主栈**：存储所有元素
2. **辅助栈**：存储每个位置对应的最小值

### 核心思想

每次 push 时，辅助栈存储当前所有元素的最小值：
- 如果辅助栈为空，直接 push 当前值
- 否则，push min(当前值, 辅助栈栈顶)

这样，辅助栈的栈顶始终是当前栈中所有元素的最小值。

### 操作流程

- **push(val)**：
  - 主栈 push val
  - 辅助栈 push min(val, 辅助栈栈顶)

- **pop()**：
  - 主栈 pop
  - 辅助栈 pop

- **top()**：返回主栈栈顶

- **getMin()**：返回辅助栈栈顶

### 示例分析

操作序列：push(-2), push(0), push(-3), getMin, pop, top, getMin

| 操作 | 主栈 | 辅助栈 | getMin |
|------|------|--------|--------|
| push(-2) | [-2] | [-2] | -2 |
| push(0) | [-2, 0] | [-2, -2] | -2 |
| push(-3) | [-2, 0, -3] | [-2, -2, -3] | -3 |
| pop() | [-2, 0] | [-2, -2] | -2 |
| top() | [-2, 0] | [-2, -2] | 0 |
| getMin() | [-2, 0] | [-2, -2] | -2 |

## 代码实现

```cpp
class MinStack {
public:
    MinStack() {
        // 构造函数不需要特殊初始化
    }
    
    void push(int val) {
        // 主栈存储元素
        mainStack.push(val);
        
        // 辅助栈存储当前最小值
        if (minStack.empty()) {
            minStack.push(val);
        } else {
            // 存储当前所有元素的最小值
            minStack.push(min(minStack.top(), val));
        }
    }
    
    void pop() {
        // 两个栈同时弹出
        mainStack.pop();
        minStack.pop();
    }
    
    int top() {
        // 返回主栈栈顶
        return mainStack.top();
    }
    
    int getMin() {
        // 返回辅助栈栈顶（当前最小值）
        return minStack.top();
    }

private:
    stack<int> mainStack;  // 主栈：存储所有元素
    stack<int> minStack;   // 辅助栈：存储每个位置的最小值
};
```

## 复杂度分析

- **时间复杂度**：所有操作都是 O(1)。
- **空间复杂度**：O(n)，需要额外的辅助栈存储最小值。

## 总结

本题使用**辅助栈**实现 O(1) 时间获取最小值。关键在于：

1. **辅助栈同步更新**：每次 push/pop 时，辅助栈同步操作
2. **存储当前最小值**：辅助栈栈顶始终是当前所有元素的最小值
3. **空间换时间**：使用额外空间换取 O(1) 时间复杂度

这是栈的经典设计问题，辅助栈的思想在很多场景都有应用。
