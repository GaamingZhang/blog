# 146. LRU 缓存

请你设计并实现一个满足  LRU (最近最少使用) 缓存 约束的数据结构。
实现 LRUCache 类：
LRUCache(int capacity) 以正整数作为容量 capacity 初始化 LRU 缓存
int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；
如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，
则应该 逐出 最久未使用的关键字。
函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

示例：
输入
["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]
[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]
输出
[null, null, null, 1, null, -1, null, -1, 3, 4]

解释
LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1); // 缓存是 {1=1}
lRUCache.put(2, 2); // 缓存是 {1=1, 2=2}
lRUCache.get(1);    // 返回 1
lRUCache.put(3, 3); // 该操作会使得关键字 2 作废，缓存是 {1=1, 3=3}
lRUCache.get(2);    // 返回 -1 (未找到)
lRUCache.put(4, 4); // 该操作会使得关键字 1 作废，缓存是 {4=4, 3=3}
lRUCache.get(1);    // 返回 -1 (未找到)
lRUCache.get(3);    // 返回 3
lRUCache.get(4);    // 返回 4

## 解题思路

这道题需要设计一个 LRU（最近最少使用）缓存，要求 get 和 put 操作都是 O(1) 时间复杂度。

### 数据结构选择

为了实现 O(1) 时间复杂度，需要结合两种数据结构：

1. **哈希表**：O(1) 时间查找 key 是否存在
2. **双向链表**：O(1) 时间插入和删除节点

### LRU 策略

- **最近使用的**：放在链表头部
- **最久未使用的**：放在链表尾部
- **淘汰策略**：当容量满时，删除链表尾部的节点

### 实现细节

1. **哈希表**：存储 key 到链表节点的映射
2. **双向链表**：维护节点的使用顺序
3. **虚拟头尾节点**：简化边界处理

### 操作流程

- **get(key)**：
  1. 在哈希表中查找 key
  2. 如果存在，将节点移到链表头部，返回 value
  3. 如果不存在，返回 -1

- **put(key, value)**：
  1. 如果 key 存在，更新 value，将节点移到头部
  2. 如果 key 不存在，创建新节点，插入头部
  3. 如果超过容量，删除尾部节点

## 代码实现

```cpp
// 双向链表节点
struct DNode {
    int key;
    int value;
    DNode* prev;
    DNode* next;
    DNode() : key(0), value(0), prev(nullptr), next(nullptr) {}
    DNode(int _key, int _value) : key(_key), value(_value), prev(nullptr), next(nullptr) {}
};

class LRUCache {
private:
    int size;       // 当前缓存大小
    int capacity;   // 缓存容量
    DNode* head;    // 虚拟头节点
    DNode* tail;    // 虚拟尾节点
    unordered_map<int, DNode*> cache;  // 哈希表：key -> 节点

public:
    LRUCache(int _capacity) : capacity(_capacity), size(0) {
        // 初始化虚拟头尾节点
        head = new DNode();
        tail = new DNode();
        head->next = tail;
        tail->prev = head;
    }
    
    int get(int key) {
        // 如果 key 不存在，返回 -1
        if (!cache.count(key)) {
            return -1;
        }
        
        // key 存在，将节点移到头部（表示最近使用）
        DNode* node = cache[key];
        moveToHead(node);
        return node->value;
    }
    
    void put(int key, int value) {
        if (!cache.count(key)) {
            // key 不存在，创建新节点
            DNode* node = new DNode(key, value);
            // 添加到哈希表和链表头部
            cache[key] = node;
            addToHead(node);
            ++size;
            
            // 如果超过容量，删除最久未使用的节点（尾部）
            if (size > capacity) {
                DNode* removed = removeTail();
                cache.erase(removed->key);
                delete removed;
                --size;
            }
        } else {
            // key 存在，更新 value 并移到头部
            DNode* node = cache[key];
            node->value = value;
            moveToHead(node);
        }
    }
    
private:
    // 将节点添加到链表头部
    void addToHead(DNode* node) {
        node->prev = head;
        node->next = head->next;
        head->next->prev = node;
        head->next = node;
    }
    
    // 从链表中删除节点
    void removeNode(DNode* node) {
        node->prev->next = node->next;
        node->next->prev = node->prev;
    }
    
    // 将节点移到链表头部
    void moveToHead(DNode* node) {
        removeNode(node);
        addToHead(node);
    }
    
    // 删除链表尾部节点（最久未使用）
    DNode* removeTail() {
        DNode* node = tail->prev;
        removeNode(node);
        return node;
    }
};
```

## 复杂度分析

- **时间复杂度**：get 和 put 操作都是 O(1)。哈希表查找 O(1)，链表插入删除 O(1)。
- **空间复杂度**：O(capacity)，哈希表和链表最多存储 capacity 个节点。

## 总结

本题使用**哈希表 + 双向链表**实现 LRU 缓存。关键在于：

1. **哈希表**：O(1) 时间查找 key
2. **双向链表**：O(1) 时间维护使用顺序
3. **虚拟节点**：简化链表边界处理
4. **淘汰策略**：删除链表尾部节点

这是经典的缓存设计问题，理解 LRU 策略和数据结构的选择是解题关键。
