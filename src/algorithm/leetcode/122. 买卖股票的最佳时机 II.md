# 122. 买卖股票的最佳时机 II

给你一个整数数组 prices ，其中 prices[i] 表示某支股票第 i 天的价格。

在每一天，你可以决定是否购买和/或出售股票。你在任何时候 最多 只能持有 一股 
 股票。你也可以先购买，然后在 同一天 出售。
返回 你能获得的 最大 利润 。

示例 1：
输入：prices = [7,1,5,3,6,4]
输出：7
解释：在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候
 卖出, 这笔交易所能获得利润 = 5 - 1 = 4。
随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候
 卖出, 这笔交易所能获得利润 = 6 - 3 = 3。
最大总利润为 4 + 3 = 7 。

## 解题思路

这道题与第 121 题不同，可以进行多次交易，但同一时间只能持有一股股票。

### 贪心算法

核心思路：**只要今天比昨天价格高，就进行交易**。

为什么这样做是正确的？

1. 如果股票价格持续上涨，如 [1, 2, 3, 4]，贪心算法会累加 (2-1) + (3-2) + (4-3) = 3，等于直接买入卖出 (4-1) = 3
2. 如果股票价格有波动，如 [1, 5, 3, 6]，贪心算法会累加 (5-1) + (6-3) = 7，这是最优解
3. 如果股票价格下跌，不进行交易，利润为 0

### 算法流程

遍历价格数组，对于每一天：
- 如果今天价格比昨天高，累加差价到利润中
- 否则跳过

## 代码实现

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int result = 0;  // 总利润
        
        // 从第二天开始遍历
        for (int i = 1; i < prices.size(); ++i) {
            // 如果今天比昨天价格高，就进行交易
            // 累加正利润
            result += max(0, prices[i] - prices[i - 1]);
        }
        
        return result;
    }
};
```

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 是数组长度。只需遍历一次。
- **空间复杂度**：O(1)，只使用了常数级别的额外空间。

## 总结

本题使用**贪心算法**，核心思想是：**收集所有正利润**。

贪心算法的正确性在于：
- 多次交易的总利润等于所有正差值之和
- 不需要考虑具体的买入卖出时机，只需要累加所有上涨的利润

与第 121 题的区别：
- 第 121 题：只能交易一次，需要找到最大差值
- 第 122 题：可以交易多次，累加所有正差值
